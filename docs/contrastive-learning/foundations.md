# 对比学习基础方法

**对比学习的核心问题非常朴素**：  
如果没有标签，模型怎么知道“哪些样本应该接近，哪些样本应该远离”？

**它的答案是**：自己构造正样本和负样本，用相似性学习一个结构化表征空间。

## 1. 从相似度函数开始

给定编码器 \(f_\theta\)，输入样本 \(x\) 的表示为：

$$
z = f_\theta(x)
$$

**常见相似度函数是余弦相似度**：

$$
\text{sim}(z_i, z_j)=\frac{z_i^\top z_j}{\|z_i\|\|z_j\|}
$$

若 \(x_i^+\) 是 \(x_i\) 的正样本，\(x_j^-\) 是负样本，则我们希望：

$$
\text{sim}(z_i, z_i^+) \uparrow,\qquad \text{sim}(z_i, z_j^-) \downarrow.
$$

### 1.1 为什么“相似度”就是核心接口

对比学习和分类最大的差别之一在于：  
分类学习的是“属于哪个类”；  
对比学习更直接地学习“谁和谁应该靠近”。  
因此整个方法族都围绕一个问题旋转：

1. 表征空间怎样定义；
2. 正负样本怎样构造；
3. 相似度分布怎样被损失函数塑形。

## 2. InfoNCE：最常见的目标函数

**最典型目标是**：

$$
\mathcal{L}_{\text{InfoNCE}}=
-\log
\frac{\exp(\text{sim}(z_i,z_i^+)/\tau)}
{\exp(\text{sim}(z_i,z_i^+)/\tau)+\sum_{j}\exp(\text{sim}(z_i,z_j^-)/\tau)}
$$

这里的温度参数 \(\tau\) 控制分布尖锐程度。  
从直觉上看，这个损失在问模型：

“在一堆候选里，你能不能把真正匹配的那个放到最前面？”

### 2.1 温度参数到底在调什么

\(\tau\) 越小，softmax 分布越尖锐，模型会更强地惩罚“没把正样本拉到最前”的情况；  
\(\tau\) 越大，分布更平缓，训练信号更柔和。

因此温度并不只是数值技巧，而是在调表征几何的“硬度”。

### 2.2 InfoNCE 的一个检索视角

如果把一个 batch 看成一个小型候选库，那么每次训练其实都在做一次检索：

1. query 是 \(z_i\)；
2. 正样本是唯一正确匹配；
3. 其余候选都是干扰项。

这也是为什么对比学习和检索系统天然联系紧密。

## 3. 正样本和负样本是怎么来的

在视觉自监督里，最常见的方法是对同一张图做两次增强：

$$
\tilde{x}_i^{(1)} = t_1(x_i), \qquad \tilde{x}_i^{(2)} = t_2(x_i)
$$

于是：

- 同一原图的两种增强视图组成正对；
- 其他图片的增强视图组成负对。

**这要求数据增强要“恰到好处”**：

- 足够强，逼模型学语义而不是像素细节；
- 但又不能强到把语义本身破坏掉。

### 一个生动例子：柴犬照片

同一张柴犬图片做随机裁剪、颜色扰动、模糊后，模型仍应知道“这是同一只狗”。  
但如果裁剪得只剩一角草地，那就不再是合理正样本，因为语义已经被抹掉了。

### 3.1 正样本构造其实定义了“什么该被保留”

若增强包含颜色抖动，模型会被鼓励忽略颜色细节；  
若增强包含随机裁剪，模型会被鼓励保留局部到整体的语义一致性。  
所以对比学习并不是无监督地“自动学一切”，而是通过增强策略显式告诉模型：

**哪些变化应该被当成不重要。**

## 4. 为什么 projection head 很关键

很多方法都会把编码器输出 \(h_i\) 再过一个投影头：

$$
h_i = f_\theta(x_i),\qquad z_i = g_\phi(h_i)
$$

然后在 \(z_i\) 上做对比损失。

### 4.1 这样做的意义

projection head 允许模型在“训练用空间”和“下游任务用空间”之间解耦：

- \(z_i\) 更适合满足对比损失；
- \(h_i\) 更适合作为下游表示。

**这解释了一个常见现象**：  
训练时用 projection head 效果更好，但下游评测时常直接丢掉它。

## 5. SimCLR：大 batch + 强增强

`SimCLR` 的核心结论非常有影响力：  
只靠大 batch、强增强和 `InfoNCE`，就能学出很强的视觉表征。

**典型流程**：

1. 对每张图生成两种增强；
2. 经过编码器得到表示；
3. 再经过一个 projection head；
4. 在投影空间上做对比损失。

### SimCLR 的启示

- 数据增强设计极其重要；
- batch 越大，负样本越丰富；
- 无监督表征学习并不一定需要复杂结构。

### 5.1 为什么大 batch 有帮助

因为在 `InfoNCE` 里，负样本数量越多，任务越接近“从更大的候选池里找正样本”。  
**这通常会带来**：

- 更强的区分性；
- 更稳定的训练信号；
- 更好的检索表现。

当然，代价就是显存和通信成本也会上升。

## 6. MoCo：用队列解决大 batch 成本

`MoCo` 观察到：如果要很多负样本，不一定非要真实大 batch，也可以维护一个动态字典队列。

**它包含**：

- 在线编码器 \(f_\theta\)；
- 动量编码器 \(f_{\bar{\theta}}\)。

**动量更新为**：

$$
\bar{\theta} \leftarrow m\bar{\theta} + (1-m)\theta
$$

其中 \(m\) 一般接近 `1`。

然后把旧 batch 的键值保存在队列中作为负样本库。  
这让模型在有限显存下也能看到大量负对。

### 一个直观理解

`SimCLR` 像是把整个班同学同时拉到教室里比对。  
`MoCo` 则像是保留了一个“过去几届学生的名册”，当前样本可以拿它来做对比，不需要每次都把所有人都叫到现场。

### 6.1 为什么需要动量编码器

如果队列里的表示来自一个变化太快的编码器，负样本库就会很不稳定。  
动量编码器的作用就是让 key 的表示更新更平滑，从而减少字典漂移。

## 7. BYOL：没有负样本也能学

`BYOL` 的结果让很多人一开始很意外：  
即使没有显式负样本，只靠在线网络和目标网络之间的一致性，也能学到不错表示。

**典型形式是**：

$$
z = g_\theta(f_\theta(x)),\qquad z' = g_{\bar{\theta}}(f_{\bar{\theta}}(x'))
$$

再让在线预测头 \(q_\theta\) 去逼近目标分支：

$$
\mathcal{L}_{\text{BYOL}} = \|q_\theta(z) - \text{sg}(z')\|_2^2
$$

这里 \(\text{sg}\) 表示 stop-gradient。

**这说明**：  
避免塌缩不一定非得靠显式负样本，也可以靠结构不对称和动量目标网络。

### 7.1 为什么不会直接塌缩

这是非对比学习最重要的问题之一。`BYOL` 常被认为依靠以下因素共同避免平凡解：

1. 在线分支和目标分支不对称；
2. stop-gradient 破坏了完全对称更新；
3. 动量目标网络让目标更稳定；
4. 预测头引入额外结构约束。

换句话说，它不是“无约束地做一致性”，而是在结构设计上偷偷加了防塌缩机制。

## 8. 对比学习到底学到了什么

从几何上看，它在学习一个嵌入空间，使得：

- 同语义实例形成局部簇；
- 不同语义实例彼此分离；
- 跨视图不变性被压进表示里。

从信息论视角看，它鼓励模型保留那些跨增强视图共享的信息，丢掉只属于单一视图的噪声。

### 一个例子：工业缺陷检测

一张金属表面照片经过裁剪和亮度扰动后，模型仍应保留“这是有划痕还是正常表面”的关键信息，而忽略光照变化和背景纹理差异。  
这就是为什么对比学习特别适合做低标注成本预训练。

### 8.1 但“学到区分性”不等于“学到任务相关性”

模型也可能学得非常会区分每个实例，却没有把对下游真正重要的因素放在首位。  
这也是为什么增强策略、负样本构造和下游评测必须一起看。

## 9. 对比学习为什么对多模态也成立

在 `CLIP` 这类模型里，正样本不再是“同一图片的两个增强”，而是“匹配的图和文本”：

$$
\mathcal{L}_{\text{clip}} = \mathcal{L}_{i\rightarrow t} + \mathcal{L}_{t\rightarrow i}.
$$

这本质上还是对比学习，只是正对换成跨模态配对。  
所以多模态对齐并不是一条全新路线，而是对比学习在图文空间里的自然延伸。

### 9.1 这件事为什么重要

因为它说明对比学习不是一个“视觉小技巧”，而是一类普适的表示学习范式。  
**只要你能定义**：

- 什么算匹配；
- 什么算不匹配；
- 表示空间如何比较；

对比学习就能自然扩展到文本、语音、视频、检索乃至机器人轨迹。

## 10. 常见失败模式

### 10.1 增强过强

把语义也裁掉了，正样本构造失真。

这类错误最容易发生在“为了追求不变性，结果把真正该保留的信息也一起抹掉”的场景。例如做细粒度分类或缺陷检测时，过强裁剪、过强模糊和颜色扰动，可能把类别差异、局部病灶或表面裂纹一起抹平。模型最后学到的不是稳健语义，而是“只要大概长得差不多都算同一个”。

因此增强设计不能只看通用经验，而要回到任务边界。对全局语义检索有利的增强，不一定适合工业检测、医学影像或遥感识别。增强一旦越过任务允许的不变性边界，正样本就不再是真正的“同一对象”了。

### 10.2 负样本其实是“假负样本”

两张不同图片可能都属于同一类别，但在无监督设定里被当成负样本。  
这会对细粒度任务造成干扰。

假负样本的麻烦在于，它不会像训练崩溃那样立刻暴露，而是悄悄把表示空间拉歪。模型会被鼓励把本来应该靠近的样本推开，例如同一类病灶的不同病例、同一产品的不同视角，或同一地块在不同季节的遥感图像。这样得到的表示可能仍然“区分性很强”，但任务相关性被破坏了。

这也是为什么很多实际系统会引入更谨慎的采样制度，比如基于元数据、时间邻近、空间邻近或弱标签的负样本过滤。对比学习不是负样本越多越好，而是**负样本越有信息越好**。

### 10.3 只学到实例区分，不学任务相关信息

模型可能非常擅长把每张图分开，却不一定学到适合下游任务的结构。

这往往发生在训练目标和业务目标差得较远时。若训练只奖励“把每个实例彼此拉开”，模型就可能把背景纹理、拍摄风格、设备噪声甚至样本 ID 一样的伪特征当成区分依据。这样在离线检索里也许表现不错，但一遇到真正的任务迁移，效果就会明显变差。

所以看对比学习结果时，不能只问“能不能分开实例”，还要问“分开的依据是不是我关心的那种差异”。如果系统真正需要的是类别、结构、区域或语义属性，那么训练和评测都要围绕这些维度补证据。

### 10.4 表征很强，但线性探测不代表全部

有时 linear probe 好看，并不代表在检索、检测、分割等任务上都同样好。

线性探测只回答一个非常有限的问题：当前表示在一个固定读取头下是否可线性分离。它并不直接回答近邻结构是否合理、跨模态是否稳、长尾样本是否可检、区域级信息是否被保留。很多表示在线性分类上漂亮，是因为全局语义分得开；但一到细粒度检索、局部检测或开放集匹配，问题马上暴露。

因此，线性 probe 更适合作为一个低成本检查项，而不是最终结论。真正成熟的验收应当把它和检索、聚类、下游任务迁移、难例桶表现一起看。

### 10.5 训练目标和部署目标错位

比如训练时只强调全局语义不变性，但上线任务更需要细粒度属性区分。

这是最工程化、也最常见的失败模式。一个团队可能在公开数据上把表示训练得很好，但真正上线时需要的是品牌 logo 区分、病灶边界识别、版面元素匹配、SKU 细粒度检索或屏幕控件识别。这些任务需要保留的，不再只是“总体像不像”，而是局部结构和关键属性。

只要训练目标没有提前对准这些需求，表示空间就会出现系统性错位。对比学习的难点，从来不只是把 loss 写对，而是让“训练时鼓励的几何关系”和“部署时真正有价值的几何关系”尽量一致。

## 11. 三个更落地的例子

### 11.1 工业缺陷检测预训练

先用大量无标签金属表面图做对比学习，再用少量标注微调分类器，通常明显优于从头训练。

这类场景的关键在于，无标签图像往往很容易收集，但真正标注裂纹、气孔、压痕、污染物等缺陷类型非常贵。对比学习先把表面纹理、形状连续性和异常局部模式学出来，再用少量标签补齐决策边界，通常会比直接从头监督学习更省标注成本。

不过这里的增广要尤其谨慎。若过强模糊或裁剪把细小缺陷抹掉，模型反而会被训练成“忽略真正重要的异常”。因此工业场景更强调缺陷敏感的增强设计，而不是通用视觉任务里的强不变性。

### 11.2 医学影像

医学数据标注贵，对比学习能先学出病灶、器官结构等稳健表示，再接小样本下游任务。

医学影像的价值在于，它天然具有丰富结构，但专家标注极贵。对比学习在这里最适合扮演“先学结构、再学标签”的角色，例如先在大量未标注 CT、MRI、病理切片上学出稳定表示，再迁移到分型、检测、分割或风险评分任务。

但医学场景也特别容易被假负样本和错误增广伤害。不同病人可能属于同一病灶模式，不适合简单互相当负样本；左右翻转、强颜色扰动或过强裁剪，也未必符合医学语义。这里最需要的不是更激进的技巧，而是更贴合成像机制和诊断逻辑的采样制度。

### 11.3 遥感图像

不同季节、不同光照、不同传感器下，同一地块外观变化很大。  
好的对比目标能帮助模型保留“地块类别与空间结构”，而不是过度依赖颜色风格。

遥感任务特别适合体现对比学习的真正价值，因为它天然要求模型跨时间、跨季节、跨设备保持一致表示。同一地块在春夏秋冬、不同卫星、不同成像条件下外观差异很大，但土地利用类型、道路结构、水体边界和建筑格局却相对稳定。对比学习若设计得好，能把这些真正稳定的结构提炼出来。

与此同时，遥感也最容易暴露“实例区分和任务相关性不一致”的问题。若模型只记住颜色风格、天气状态或传感器噪声，那么表示会在离线看起来很丰富，却对地块分类、变化检测和时序检索帮助有限。

## 12. 一个实用判断框架

如果你在设计对比学习系统，至少要先问四个问题：

1. 正样本构造到底保留什么不变性；
2. 负样本里有多少假负；
3. 训练时的表征空间和下游使用空间是否一致；
4. 最终目标是检索、迁移、鲁棒性，还是细粒度识别。

很多所谓“方法优劣”最后都回到这四个问题，而不只是某篇论文里换了一个损失写法。

## 13. 总结

对比学习最重要的贡献，不只是一个损失函数，而是一种训练范式：

- 自己定义什么是“相似”；
- 自己构造监督信号；
- 再把表示空间塑造成适合下游任务的几何结构。

从 `SimCLR` 到 `MoCo`，再到 `BYOL` 和 `CLIP`，这条路线不断在回答同一个问题：  
**没有标签时，模型如何仍然学到对世界有用的表示。** 

## 快速代码示例

```python
import torch
import torch.nn.functional as F

def simclr_step(encoder, x1, x2, tau=0.07):
    z1 = F.normalize(encoder(x1), dim=-1)
    z2 = F.normalize(encoder(x2), dim=-1)
    logits = z1 @ z2.T / tau
    labels = torch.arange(z1.size(0), device=z1.device)
    return F.cross_entropy(logits, labels)
```

这段代码对应 **SimCLR** 的核心训练步：同一批样本两次增强后得到 `z1/z2`，通过温度缩放的相似度矩阵构造对比目标。它的重点是让同一样本的两种视图靠近，同时把其他样本推远。


## 补充：把 **对比学习基础方法** 从损失函数写成表示系统

对比学习页面若只讲公式和论文脉络，读者很容易把它理解成一类训练技巧；但真正更有价值的视角是：**它在组织一个表示系统**。围绕 **对比学习基础方法**，更扎实的页面应同时说明表示几何、采样制度、增广与负样本策略、下游读取方式，以及线上检索或多模态系统如何真正消费这些表示。

从这个角度看，从目标函数、温度、负样本制度、迁移收益和失败模式理解基础方法。因此，对比学习的关键从来不只是“损失写成什么样”，还包括：哪些样本被拉近、哪些样本被推远、这种几何关系是否真的符合任务边界、以及这种关系在部署后会不会因为分布漂移而崩掉。

## 补充：从训练目标到下游消费的一条链

围绕 **对比学习基础方法**，建议沿着下面这条链来理解：

1. **训练目标** 决定表示空间希望保留哪些不变性；
2. **采样与增广制度** 决定模型究竟会把什么当作“同一个对象”或“不同对象”；
3. **评测与读取方式** 决定这种表示是否真的能被检索、聚类、重排、多模态对齐或下游任务消费；
4. **线上分布变化** 决定这些几何关系是否仍然成立。

只要把这条链讲明白，很多常见误解就会自然消失。比如为什么某些表示在离线线性评估里很好，但线上检索或跨模态系统里收益有限；为什么难负样本设计稍有不慎就会把本来该靠近的东西推开；为什么 teacher-student 或 non-contrastive 方法在某些场景下更稳，而在另一些场景里又会被长尾和分布偏移放大缺陷。

## 补充：更常见、也更隐蔽的失败模式

对比学习最容易被低估的风险，是它常常在“整体上看起来没问题”时悄悄损伤任务边界。比如表示过度压缩导致难例分不清，过强增广导致语义信息被抹掉，负样本制度不合适导致相关样本被错误推远，或者多模态对齐表面很好、实际线上文本和图像分布漂移后迅速失效。

这类失败之所以难，是因为它们很少通过一个总分暴露。页面若要更扎实，就应明确告诉读者：**一定要按任务桶、难例桶、线上流量桶来看表示质量**，而不是相信一个总体召回率或线性 probe 分数就足够。

## 补充：更像工程文档的验收方式

对 **对比学习基础方法**，更可靠的验收至少应同时看：
