# 数据发布检查清单

数据集一旦发布，不只是“把文件放出去”这么简单。它会定义后续训练、评测、复现和合规边界。如果数据版本说明不清、标注协议含糊、授权边界不明、长尾样本缺失、评测切分污染，那么后续整个研究链条都可能建立在不稳的地基上。这份清单适合内部数据版本冻结、对外数据发布、共享标注集移交和 benchmark 建设。

## 1. 数据身份信息

**发布前先确认**：

1. 数据集名称与版本号
2. 发布时间
3. 负责人
4. 数据来源范围
5. 预期用途与不适用场景

没有这一步，后续引用、复现和问题追踪都会混乱。

## 2. 来源与授权

**必须写清**：

1. 数据来自公开采集、合作授权、内部生成还是用户回流；
2. 是否允许训练、评测、再分发；
3. 是否存在地域、行业或商业限制；
4. 是否包含第三方受限素材。

尤其是多模态和企业数据，授权边界不清是最危险的隐患之一。

## 3. 数据结构与字段说明

**发布文档至少应包含**：

1. 文件组织结构
2. 样本字段定义
3. 坐标、时间戳、语言、版本等元数据说明
4. 缺失值、异常值和默认值规则

对于文档、VLM、VLA 和机器人数据，还要特别说明模态之间如何对齐。

## 4. 标注协议

**需要明确**：

1. 标签定义
2. 标注工具
3. 标注流程
4. 质检规则
5. 多标注员冲突如何解决

如果评测或训练依赖复杂标签，而协议没有公开，数据价值会大幅下降。

## 5. 切分与污染检查

确认训练 / 验证 / 测试切分逻辑，尤其要检查：

1. 是否存在重复或近重复样本跨集合出现；
2. 是否存在同文档不同页、同视频不同片段泄漏；
3. 是否存在同任务轨迹跨集合污染；
4. 是否有 benchmark 与训练集交叉。

必要时记录近重复阈值和去重方法。

## 6. 质量统计

**发布前建议产出最基本统计**：

1. 样本总量
2. 各类别分布
3. 长尾比例
4. 分辨率 / 长度 / 时长分布
5. 低质量样本比例

没有这些统计，使用者很难判断数据适合什么问题。

## 7. 风险与限制

**数据集文档应显式写出**：

1. 已知偏置
2. 覆盖盲区
3. 不建议使用场景
4. 潜在安全或伦理风险

一个诚实的数据集文档，价值远高于只写优点的宣传说明。

## 8. 评测建议

若数据同时服务 benchmark，建议提供：

1. 推荐主指标
2. 分桶方式
3. 长尾评测注意事项
4. 不应比较的口径

否则不同团队很可能在不同评测口径下得出不可比较的结果。

## 9. 发布包完整性

**最终发布时应检查**：

1. 数据文件完整
2. 校验和可用
3. 示例读取脚本可运行
4. 许可证文件存在
5. README 与 schema 对齐

## 10. 小结

数据发布清单的意义，是让数据从“可分享文件”变成“可复用资产”。一份好的数据版本，不只要能下载，更要能被正确理解、合规使用、稳定复现，并在后续研究中持续产生价值。
