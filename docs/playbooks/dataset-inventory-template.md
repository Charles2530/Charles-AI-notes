# 数据集盘点模板

这一页适合在项目启动或数据治理阶段，对数据资产做一次系统盘点。目标不是只列文件名，而是回答：

- 数据到底来自哪里
- 覆盖了哪些场景
- 哪些风险还没处理
- 这份数据现在到底能不能支撑当前任务

很多团队的问题不是“没有数据”，而是“有很多数据，但没人能一句话说清它们是什么、适不适合当前任务”。这份模板的价值就在这里。

## 1. 基本信息

```text
数据集名称：
负责人：
版本：
时间范围：
主要用途：
```

**建议额外记录**：

```text
服务对象 / 使用团队：
下游任务：
预期发布范围（内部 / 对外 / 仅评测）：
当前状态（草稿 / 冻结 / 已发布）：
```

把这些信息写清楚，有助于后续判断这份数据到底是训练集、评测集、回流池，还是仅供探索的原始资产。

## 2. 数据来源

**建议按来源逐项列出**：

- 内部日志
- 开源数据
- 标注数据
- 合成数据
- 仿真数据

如果是混合数据，最好写：

\[
\mathcal{D} = \bigcup_{k=1}^{K}\mathcal{D}_k
\]

并记录各部分比例。

### 建议补充的来源字段

对每个 \(\mathcal{D}_k\) 建议至少记录：

1. 来源名称；
2. 获取方式；
3. 授权状态；
4. 时间范围；
5. 是否含敏感信息；
6. 是否依赖外部版本或工具链。

### 一个可直接复用的来源表

| 来源 | 类型 | 样本量 | 时间范围 | 授权状态 | 备注 |
|---|---|---:|---|---|---|
| 内部日志 | 真实流量 |  |  |  |  |
| 开源数据 | 公共语料 |  |  |  |  |
| 标注数据 | 人工高质量 |  |  |  |  |
| 合成数据 | 模板 / 模型生成 |  |  |  |  |
| 仿真数据 | 环境生成 |  |  |  |  |

## 3. 覆盖范围

**建议明确记录**：

- 场景覆盖
- 类别覆盖
- 长尾覆盖
- 时间覆盖
- 语言覆盖

不要只写“数据很多”，而要写“覆盖了什么、没覆盖什么”。

### 覆盖范围最好拆成两个层面

1. `主流覆盖`：最常见场景是否足够；
2. `长尾覆盖`：系统最容易失败的边界是否被看到。

例如对文档 VLM，可进一步拆成：

1. 文档类型覆盖；
2. OCR 质量覆盖；
3. 表格 / 图表 / 手写 / 盖章覆盖；
4. 多语言覆盖；
5. 低质扫描与异常版式覆盖。

对 VLA，可进一步拆成：

1. 任务类型；
2. 物体种类；
3. 光照与相机位姿；
4. 失败恢复；
5. 人工接管和 near-failure 样本。

## 4. 质量与噪声

**建议记录**：

- 重复率
- 标注噪声
- 缺失字段
- 图像/文本质量
- 分布偏差

### 建议增加可量化统计

若条件允许，最好补：

1. 近重复率；
2. 标注一致性；
3. OCR / ASR 错误率；
4. 低分辨率占比；
5. 结构错配样本比例；
6. 损坏文件或无效样本比例。

### 一个关键判断问题

**请明确回答**：

`这份数据里，最可能让模型学偏的噪声是什么？`

这往往比机械罗列更多统计更能帮助后续训练决策。

## 5. 合规与权限

**至少要写清楚**：

- 数据授权状态
- 是否可商用
- 是否含敏感信息
- 是否需要脱敏

若是内部或真实用户数据，建议额外写：

1. 是否允许训练；
2. 是否允许评测；
3. 是否允许被外部服务处理；
4. 是否有删除请求与审计机制；
5. 是否受地域或行业监管约束。

## 6. 风险与缺口

**建议固定写**：

1. 当前最大数据缺口是什么？
2. 哪些关键场景覆盖不足？
3. 哪些字段最容易引起误导？

### 建议把缺口分级

**可以按优先级写**：

1. `P0`：不补就会影响当前任务上线或评测可信度；
2. `P1`：会明显限制下一轮模型收益；
3. `P2`：长期需要补，但当前不构成立即阻塞。

这样盘点文档就能自然转成行动计划，而不是停留在“描述现状”。

## 7. 结论

**最终应能回答**：

- 这份数据适不适合当前任务
- 是否需要补采或清洗
- 下一轮最优先补哪一块

### 推荐的结论格式

```text
当前判断：
可直接用于：
不可直接用于：
最优先动作：
负责人：
预计完成时间：
```

## 8. 一个简短示例

假设你在盘点一份“企业文档问答”数据集，一个真正有用的结论通常不是“共有 200 万页文档”，而应是：

1. 合同和报销单主流样本充足；
2. 低质扫描 PDF 覆盖不足；
3. 表格页和盖章页标注不一致；
4. 一部分权限文档尚未完成脱敏；
5. 下一轮应优先补复杂表格与扫描 hard set。

只有写到这个粒度，盘点才会真正指导后续训练和评测。
