# 实验记录模板

这一页给出一个偏研究和建模侧的实验记录模板。  
**它适合用于**：

- 模型训练实验
- 消融实验
- 采样器比较
- 量化与部署前评估

## 1. 基本信息

```text
实验名称：
日期：
负责人：
模型/代码版本：
数据版本：
运行环境：
```

## 2. 实验目标

建议明确写成一句可验证的话，例如：

- 验证 `DPM-Solver++` 在 15 步下是否优于 `Euler`
- 评估 `AWQ 4bit` 对长上下文问答的影响
- 比较不同 LoRA rank 在医疗问答上的收益

避免写成过于模糊的目标，例如“看看效果怎么样”。

## 3. 问题定义

若实验涉及明确优化目标，可以写成：

\[
\min_\theta \mathcal{L}(\theta)
\]

或：

\[
\max_\pi \mathbb{E}[R]
\]

若是系统实验，也可写成：

\[
\min \text{latency}, \qquad \max \text{quality}
\]

关键不是形式主义，而是把“到底在优化什么”写清楚。

## 4. 假设

建议把假设单列，例如：

- 假设 4bit 权重量化对短问答影响小，但会显著影响长链推理
- 假设高 guidance 更依赖高阶 solver
- 假设失败主要来自 OCR，而不是 rerank

**这样后面复盘时才能判断**：

- 是假设错了
- 还是实现没跟上

## 5. 实验设置

```text
模型：
训练/推理配置：
数据范围：
随机种子：
batch size：
学习率：
步数/epoch：
评测集：
主要超参：
```

如果是扩散实验，建议额外写：

```text
采样器：
步数：
guidance scale：
noise schedule：
```

如果是服务实验，建议额外写：

```text
并发数：
输入长度分布：
输出长度分布：
硬件：
```

## 6. 指标

按任务类型拆开写，不要只写一个总分。

### 建模指标

- loss
- perplexity
- FID
- accuracy
- success rate

### 系统指标

- TTFT
- TPOT
- P95 latency
- GPU memory
- throughput

### 业务指标

- 点击率
- 工单完成率
- 人工接管率

## 7. Baseline

**一定要写清楚对照组**：

```text
Baseline A：
Baseline B：
本实验相对谁比较：
```

没有 baseline，很多“提升”没有意义。

## 8. 结果记录模板

```text
实验结果概述：

主要指标：
- 指标 1：
- 指标 2：
- 指标 3：

与 baseline 对比：
- 优于：
- 持平：
- 低于：
```

如果是可视化实验，可补：

- 代表性成功样本
- 代表性失败样本
- 边界样本

## 9. 失败样本分析

**建议强制写三类失败**：

1. 明显失败
2. 边界失败
3. 伪成功

其中“伪成功”尤其重要，例如：

- 文本答案看起来顺，但证据不对
- 图片生成风格对了，但结构错了
- 机器人完成任务了，但轨迹极不稳定

## 10. 原因判断

**建议把问题强制归类到下列之一**：

- 数据问题
- 优化问题
- 模型结构问题
- 推理配置问题
- 评测问题
- 系统实现问题

这样复盘会更清楚，不容易把所有锅都甩给“模型不够强”。

## 11. 结论

**推荐固定回答四个问题**：

1. 最初假设是否成立？
2. 最大收益来自哪里？
3. 最大退化来自哪里？
4. 是否值得进入下一轮实验？

## 12. 下一步动作

建议写成明确行动项，例如：

- 扩大校准集到真实长文档流量
- 把 LoRA rank 从 16 提到 32
- 对失败样本补 OCR 子系统日志
- 将 30 步采样器统一到 15 步评测框架

## 13. 一句经验

实验记录最怕写成流水账。  
真正有价值的记录，不是把参数都抄一遍，而是让后来的人能快速回答：

- 你试了什么
- 为什么试
- 哪些地方真有用
- 哪些地方以后别再踩
