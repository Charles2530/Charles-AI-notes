# 多模态与具身论文导读

这份导读覆盖一条从表征学习到 VLM、再到 VLA 与具身智能的阅读路径。  
它的目标不是把所有工作列齐，而是建立一条能解释“为什么这些方向会自然连起来”的主线。

## 你最终应该建立的主线

```text
对比学习
 -> 图文对齐
 -> VLM
 -> 工具使用与代理
 -> VLA
 -> 世界模型
 -> 具身智能与 sim2real
```

## 第一阶段：先建立表征学习与对齐直觉

### 1. SimCLR / MoCo / BYOL

读这几篇的目标不是背细节，而是建立一个基本认识：

- 没有标签也能学表示
- 表示空间的几何结构可以由训练目标塑造

如果这一步没建立，后面的 CLIP 会显得像“突然冒出来的图文神经网络”。

### 2. CLIP

这是多模态阅读的第一个关键节点。  
读 CLIP 时重点不是只看它能不能做零样本分类，而是理解：

- 图像和文本为什么能共享向量空间
- 对比学习为什么天然适合跨模态对齐
- 为什么检索和多模态召回会从这里起飞

## 第二阶段：进入 VLM

### 3. Flamingo / BLIP-2 / LLaVA 一类工作

这里推荐的读法不是逐篇抠 benchmark，而是比较三种问题：

1. 如何把视觉 token 接到语言模型上？
2. 冻结 LLM 和训练连接器分别带来什么好处？
3. 模型是更偏检索、问答，还是代理？

**读完后你应当能区分**：

- 双塔式图文对齐
- 视觉编码器 + LLM
- 更强调 few-shot 或 instruction tuning 的路线

## 第三阶段：从“会看会说”走向“会调用工具”

### 4. 具工具使用或屏幕操作能力的多模态代理工作

这里的阅读重点不在单个 benchmark，而在一个结构变化：

\[
\text{感知} \rightarrow \text{推理} \rightarrow \text{动作/工具}
\]

也就是说，模型输出不再只是答案，而是操作链路的一部分。

读这类工作时，要特别关注：

- 中间状态有没有显式表示
- 工具调用是否可回退
- 错误是出在感知还是规划

## 第四阶段：进入 VLA

### 5. RT-1

这是 VLA 阅读中的一个好起点。  
**重点是理解**：

- 机器人动作也可以被看作序列建模问题
- 视觉与语言条件如何进入策略
- 动作离散化为什么是有吸引力的工程选择

### 6. RT-2

RT-2 的价值在于把“网络知识和语言知识”往机器人动作中迁移。  
**读它时值得思考**：

- VLM 和 VLA 的边界在哪里
- Web-scale knowledge 到控制动作之间，真正可迁移的是什么

### 7. SayCan / PaLM-E 一类工作

读这类工作时，关键不是记住模型名字，而是理解高层语言规划与底层可执行技能之间的接口设计。

## 第五阶段：世界模型与前瞻能力

### 8. World Models / RSSM / Dreamer

**这条线回答的是**：

智能体能否先在内部模拟未来，再决定怎么动？

如果你只读 VLA，不读世界模型，就容易把具身系统理解成纯反射式策略。  
而一旦补上 Dreamer 这条线，你会更清楚：

- 表征学习
- 潜空间建模
- 规划与控制

为什么最终会在智能体系统里汇合。

## 第六阶段：sim2real 与真实部署

### 9. 域随机化、system ID、闭环数据引擎

这部分不一定只靠一篇代表论文，而更像一类工程主线。  
你应该重点读的不是“方法名字多高级”，而是：

- 为什么仿真成功不等于真实成功
- 为什么失败样本比成功样本更值钱
- 为什么真实机器人系统最终依赖闭环数据引擎

## 推荐最小阅读包

如果只想快速建立主线，建议最少读：

1. SimCLR 或 MoCo
2. CLIP
3. BLIP-2 或 LLaVA
4. RT-1
5. RT-2
6. Dreamer
7. 一篇 sim2real / 数据引擎类代表工作

## 读这条线时最值得记的四件事

1. 这个方法解决的是表征、对齐、策略，还是部署？
2. 它输出的是答案、工具动作，还是物理动作？
3. 它面对的核心瓶颈是数据、模型还是系统？
4. 它离真实机器人部署还有多远？

如果你一直用这四个问题做笔记，多模态和具身这条线就不会显得支离破碎。
