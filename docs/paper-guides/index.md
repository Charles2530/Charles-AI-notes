# 论文导读总览

这一组页面不是简单列论文清单，而是回答一个更实用的问题：

**如果你现在要进入某个主题，先读什么、为什么读、读完要带走什么。**

## 为什么要有导读页

对于新主题，很多人第一反应是搜索一串论文标题，然后按年份硬啃。  
这往往低效，因为：

- 你不知道哪篇是分水岭
- 你不知道哪些工作是主线、哪些只是分叉
- 你很容易在实现细节里迷失，忽略方法之间的继承关系

导读页的作用，就是把“阅读顺序”本身结构化。

## 导读页怎么看

建议按以下四个问题来读每篇论文：

1. 这篇论文到底解决了什么旧问题？
2. 它改的是训练、推理，还是路径设计？
3. 它和上一代方法的核心差异是什么？
4. 它留下了哪些后来工作继续解决的问题？

如果能回答这四个问题，你读论文时就不容易陷入“只记住标题，不知道位置”的状态。

## 一种推荐的使用方式

如果你的目标是做项目而不是只读论文，建议使用“导读页 + 正文页”双层方式：

- 先用导读页建立方法地图
- 再回到各主题正文页补公式、例子和工程细节

这样读会比直接在正文里来回跳转高效很多。
