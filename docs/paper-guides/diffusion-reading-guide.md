# 扩散模型论文导读

这份导读不是要列尽所有扩散论文，而是给出一条足够稳定、能快速建立主线认知的阅读路径。  
如果你按这条顺序读，最后应该能回答三个问题：

1. 扩散模型为什么有效？
2. 为什么会从 DDPM 走到 ODE solver？
3. 为什么最终又会走向蒸馏、整流和一步生成？

## 第一阶段：先读起点，理解基本建模

### 1. DDPM

**论文**：Ho, Jain, Abbeel, 2020  
读这篇的目标不是记住所有公式，而是搞清楚：

- 前向加噪过程如何定义
- 反向去噪为什么可学
- 为什么训练可转化为噪声回归

**读完后你至少应能写出**：

\[
x_t = \sqrt{\bar{\alpha}_t}x_0 + \sqrt{1-\bar{\alpha}_t}\epsilon
\]

并理解这条式子为什么让训练变成监督学习。

真正建议精读的部分，是前向过程、反向过程和噪声回归目标之间的连接方式。你不一定要在第一次阅读时把每个变分项都推完，但至少要把“为什么一个看似复杂的生成问题最后变成了带噪图监督学习”这件事彻底看懂。后面所有扩散方法，几乎都建立在这个转写之上。

### 2. DDIM

读这篇不是为了学一个新模型，而是为了建立一个关键直觉：  
扩散采样不一定非得严格沿着原始随机链走。

**读完后要理解**：

- 非马尔可夫采样路径意味着什么
- 为什么不重训也能减少步数
- 它和后续 ODE 视角之间的桥梁关系

读 `DDIM` 时，最值得盯住的不是具体实验表，而是它在观念上把扩散从“只能沿原始随机链采样”解放了出来。你应该在这一阶段明确意识到：训练好的扩散模型，并不只对应一种采样轨迹。这一点一旦建立，后面看到 solver 家族时就不会觉得它们只是换名字的工程 trick。

## 第二阶段：进入连续时间与数值求解视角

### 3. Score SDE / Probability Flow ODE

这是扩散理论视角升级的关键节点。  
你读这篇时不必一开始就抠所有推导，但必须抓住一件事：

**扩散采样可以被统一进连续时间 SDE/ODE 框架。**

如果这个视角不建立，后面的 Euler、Heun、DPM-Solver 看起来就只是采样器名字堆砌。

这篇建议分两遍读。第一遍先只抓住“为什么有 SDE，也为什么存在对应的 ODE”这个大框架；第二遍再回头看 score matching、逆时间动力学和边缘分布一致性。若第一次就陷在细节里，通常很容易被符号淹没。

### 4. EDM

这篇非常适合用来建立工程直觉。  
**它帮助你把**：

- 噪声参数化
- 训练设计
- 采样器选择

放在一个统一实验框架里比较。

如果你只读理论论文不读 EDM，很容易无法形成“不同设计项到底怎样影响质量和速度”的直觉。

`EDM` 在导读中的价值，非常像一份设计空间地图。它会让你开始把噪声参数化、采样器、训练分布和评测方式当成同一个系统来理解，而不再把每篇论文都看成只改了一个小部件。对想真正做实验的人来说，这一页通常比单看方法论文更能建立工程判断力。

## 第三阶段：理解采样器为什么会成为主线

### 5. DPM-Solver / DPM-Solver++

读这两篇的重点不是具体实现细节，而是：

- 为什么 diffusion ODE 值得专门做 solver
- 高阶求解器为什么能在 10 到 20 步仍保持质量
- classifier-free guidance 为什么会改变 solver 设计

到这一步，你应能清楚区分：

- 模型训练目标
- 推理时的数值求解器

这是扩散学习中一个非常关键的边界。

读这一组工作时，建议把它们和 `DDIM` 明确区分开。`DDIM` 更像重新解释采样路径，`DPM-Solver` 则是在明确接受 ODE 视角后，开始系统地问“扩散 ODE 到底该怎么高效积分”。一旦这个边界建立，后面你会更自然地区分“模型能力提升”和“推理端兑现效率提升”。

## 第四阶段：从“更快采样”走向“重训少步模型”

### 6. Progressive Distillation

这篇是少步蒸馏的起点之一。  
它最值得带走的不是具体算法，而是一个范式：

**让学生学会教师多步合成的一步。**

这篇最好带着“为什么 solver 再强也不够”去读。它意味着研究开始承认：推理端优化不是万能的，如果目标是极少步乃至一步生成，就必须把训练目标和学生模型一起纳入设计。这个转折点在扩散路线里非常关键。

### 7. Consistency Models / LCM

读这一组工作时，重点理解：

- 少步生成不一定必须模仿整条原始扩散轨迹
- 也可以直接学习一种更短、更一致的映射

到这一步，扩散研究已经明显从“求解器优化”进入“模型结构性加速”。

这里建议重点做一件事：把 `Progressive Distillation`、`Consistency` 和 `LCM` 放在一张表里，比较它们到底是在模仿轨迹、学习一致映射，还是直接为少步推理而重写训练目标。这样后面看一步生成路线时，脑子里会有一个很清楚的坐标系。

## 第五阶段：一步生成与分布匹配

### 8. DMD

这篇标志着一步扩散蒸馏的一个重要转向。  
**阅读重点**：

- 为什么“模仿轨迹”不够
- 为什么转向“分布匹配”
- 一步生成为什么更容易出现模糊与塌缩

`DMD` 最值得精读的地方，是它在观念上从“学生学老师怎么走”变成了“学生直接学应该落在哪个分布上”。这是扩散蒸馏理解上的一次大转弯。即使你暂时不打算做一步生成，也建议把这一步看懂，因为它会重新组织你对蒸馏目标的理解。

### 9. DMD2

它不是简单续作，而是在正面修补 DMD 的主要问题。  
读这篇时，要重点比对：

- 训练稳定性
- 图像锐度
- 生成器与假样本分布估计器的关系

读 `DMD2` 时，不要把它当成“把 DMD 再做强一点”的小修补，而要明确它在回答：一步生成真正卡住的，是不是训练信号本身不够稳、不够锐、也不够像真实分布。只要带着这个问题去看，它的改动会比论文名字显得重要得多。

### 10. Phased DMD

这篇最有价值的阅读角度是“分阶段蒸馏”的直觉。  
**也就是**：

- 早期噪声阶段负责大结构
- 后期阶段负责细节

如果你把整个过程硬压成一步，难度会很高。

这篇的阅读重点，不只是“又多了一个一步路线”，而是它把阶段化学习重新带回了极少步蒸馏。也就是说，研究者开始承认：即使目标是一两步生成，训练时仍然可能需要把不同噪声区间的职责拆开学。这对理解为什么一步生成如此难很有帮助。

## 第六阶段：整流流与 rectified 路线

### 11. Rectified Flow

重点理解的是“更直的生成路径是否更适合少步离散”。

读 rectified 路线时，最容易犯的错误是把它当成与扩散完全无关的新家族。更稳的读法是把它看成对同一个问题的另一种回答：如果极少步离散很难，那能否先把路径本身改得更适合粗离散？只要带着这个问题去读，这条线就会自然和扩散主线接上。

### 12. Rectified Diffusion

这篇值得读，是因为它在反思一个常被过度简化的观点：  
不是“越直越好”，而是“越适合粗离散越好”。

这能帮助你把 rectified 路线和传统 diffusion 路线放到同一张图里看，而不是把它们当成完全无关的方法。

`Rectified Diffusion` 的价值，正在于它提醒你不要把“直”当成教条。真正重要的是路径是否更适合少步积分、是否更容易稳定训练、是否更容易兼顾条件控制与质量。这个视角会让你读方法时更少陷入口号，多回到设计目标。

## 推荐阅读顺序汇总

如果时间很紧，最小阅读包可以是：

1. DDPM
2. DDIM
3. Score SDE
4. EDM
5. DPM-Solver++
6. Progressive Distillation
7. DMD
8. DMD2
9. Rectified Diffusion

如果你是第一次系统读扩散，建议把这九篇分成三周或三轮来读，而不是一天扫完。第一轮只建立主线，第二轮做方法对照表，第三轮再把关键公式和实验结论抄成自己的笔记。扩散导读最怕“一次看了很多，但一周后只剩标题”。

## 读完后你应该能形成的主线

```text
DDPM
 -> DDIM
 -> Score SDE / Probability Flow ODE
 -> Euler / Heun / DPM-Solver
 -> Distillation / Consistency
 -> DMD / DMD2 / Phased DMD
 -> Rectified Flow / Rectified Diffusion
```

## 阅读时最值得做的笔记

**建议每读一篇都固定记四件事**：

1. 它改的是训练、采样器，还是生成路径？
2. 它相比上一代方法快在哪、稳在哪？
3. 它最明显的副作用是什么？
4. 它留下了哪些后来论文继续补的问题？

只要这四条持续记，扩散论文不会再变成一堆互相无关的名字。

如果想把笔记再做得更能复用，建议额外加一列：**这篇更像训练改进、推理改进，还是路径改进**。只要这列写清楚，后面做方法选型时会非常省力，因为你能立刻知道某篇论文到底动了系统哪一层。


## 补充：把 **扩散模型论文导读** 写成真正可执行的阅读路径

导读页最容易的问题，是“列得很全，但读者读完仍然不知道该先看什么、为什么看、读到哪一步算过关”。围绕 **扩散模型论文导读**，更有价值的写法是把导读变成一种执行框架，而不只是导航页。

更具体地说，把扩散阅读路径写得更像真实学习计划。这意味着导读页不应只给论文名，还应告诉读者：应该先建立哪几个核心问题意识，哪些论文负责打开视野，哪些论文负责建立方法主线，哪些系统论文负责让抽象概念落到真实工程里，哪些内容适合快速跳读，哪些地方最值得做复现和做笔记。

## 补充：更像学习计划的组织方式

更成熟的导读页可以按三条线并行组织：

1. **问题线**：这个方向最本质的问题是什么；
2. **方法线**：主流方法是如何一步步逼近这些问题的；
3. **工程线**：哪些结论一旦落到系统、数据、评测和部署上会发生变化。

一旦这三条线写清楚，读者就不容易陷入“知道很多论文标题，但仍然没有判断力”的状态。尤其对系统、世界模型、VLM/VLA 这类主题，真正值钱的往往不是读过多少论文，而是能不能把论文放回完整问题空间里判断。

## 补充：导读页也应明确复现与跳读策略

很多读者最需要的其实不是再多十篇论文，而是知道：

1. 哪几篇适合精读；
2. 哪几篇只需抓核心图表和结论；
3. 哪几篇如果不看源码或系统实现就很难真正理解；
4. 哪些部分最适合做复现、做对照实验或做总结表。

只要把这些策略写进导读页，**扩散模型论文导读** 就会从“目录”升级成“真正能帮人建立学习路径的工具页”。


## 补充：导读页最该补的是接口说明

围绕 **扩散模型论文导读**，真正高价值的扩写不是继续堆论文，而是把 论文主线、方法边界和工程落地的接口 讲清楚。这样读者才知道：看完这页后，应该跳到哪一类方法页、系统页或评测页，才能把阅读路径真正闭环起来。

## 补充：一条更实用的执行顺序

更好的导读页应明确告诉读者：先看哪些奠基问题，再看哪些代表性方法，再看哪些工程化页面，最后在哪些位置做复现或做总结表。只要这条顺序写清楚，导读页就不再只是入口，而会真正变成学习计划。

## 补充：后续最值得继续补强的内容

导读页后续最值得加厚的，通常不是再加一串参考文献，而是补：跳读策略、复现优先级、经典误读、系统化做笔记的方法，以及不同背景读者各自最短的入门路径。


## 补充：如果要把 **扩散模型论文导读** 真正推进成主线工作

一个很现实的问题是：团队即使认同 **扩散模型论文导读** 很重要，也未必知道应该先投入哪一层。更稳的推进方式通常不是一上来就把所有可能相关的方法和组件都拉满，而是先判断 **它当前最可能成为主线收益点，还是主线阻塞点**。若它更像收益点，重点是尽快做小规模验证、建立对照组和收益归因；若它更像阻塞点，重点则应放在接口澄清、失败样本回放、基础设施补齐和回退路径准备上。把这一步判断写进页面，读者会更容易把“知道这个方向”转成“知道现在该做什么”。

## 补充：比继续堆内容更重要的，是明确暂时不做什么

很多方向越写越散，根因不是信息不够，而是没有明确边界。对 **扩散模型论文导读** 来说，一份更成熟的文档通常还要告诉读者：在当前阶段，哪些复杂方法可以先不碰，哪些收益看起来诱人但短期不值得，哪些局部最优会把系统拖进更高维护成本。只要把“暂时不做什么”写明白，页面的决策价值会大幅提高，因为真实项目里最贵的往往不是少做了一件事，而是过早做了十件还没有基础条件支撑的事。

## 补充：什么算这页真正写扎实了

判断 **扩散模型论文导读** 是否已经足够扎实，可以用一个很朴素的标准：读完之后，读者是否能够明确说出这件事在系统里的位置、推进顺序、主要风险、回退方案和下一步最值得补的数据或实验。如果这几个问题还能答得含糊，说明页面仍然偏“知道名词”；如果已经能答得很具体，那它才真正开始具备方法文档和工程文档的双重价值。


## 补充：快速检查清单

阅读或使用 **扩散模型论文导读** 时，至少快速核对五件事：

1. 这页讨论的是**基础方法**、**系统接口**还是**上线验收**；
2. 这里的结论更适用于**离线实验**还是**真实部署**；
3. 文中的收益依赖的是**更多数据/算力**，还是依赖**更好的接口与评测**；
4. 如果结论不成立，最先该回头检查的是**数据**、**方法**、**系统路径**还是**评测口径**；
5. 这页和相邻页面之间，哪一页负责提供下一步证据。

只要这五件事能在读完后答清楚，**扩散模型论文导读** 就不再只是资料页，而会真正成为可执行的工作页。


## 补充：读完这页后最好能明确回答

对于 **扩散模型论文导读**，读完后最好至少能回答三件事：它在整条系统链路里解决哪一段问题；它最怕在哪些数据、负载或评测口径下被误读；以及如果现在就要开工，第一步应从实验、接口、回放还是基础设施哪一侧下手。只有这三件事答清楚，页面内容才真正具备行动价值。


## 补充：项目推进提醒

如果 **扩散模型论文导读** 已经进入实际项目，请优先把样本边界、评测桶、回退条件和相关页面的接口先写清，再继续扩方法细节。多数返工都不是因为信息不够，而是因为这四件事没有先落地。


## 补充：不要只看平均值

对 **扩散模型论文导读** 来说，真正决定成败的往往不是平均指标，而是高价值任务、长尾样本、极端长度、异常接口或高风险场景下是否仍然成立。只要把这条提醒放在页面里，读者就更不容易把一组平滑平均值误当成已经足够可靠的结论。


## 补充：实践中最先补证据

对 **扩散模型论文导读** 来说，最先该补的不是更多参考文献，而是更固定的学习证据：

1. **一张主线对照表**：每篇论文改了训练、采样器还是生成路径；
2. **一组最小复现对象**：例如只复现 `DDIM`、`DPM-Solver++`、`DMD2` 三个关键转折点；
3. **一份跳读清单**：哪些论文先看图和结论，哪些必须读方法节；
4. **一份误区清单**：哪些工作最容易被读成“更快就是更好”。

有了这些证据，导读页才真的能帮助学习，而不是只负责导航。

一旦 **扩散模型论文导读** 进入主线，优先补的通常不是更多概念，而是更多**可比较证据**：固定样本、固定回放、固定桶和固定回退门槛。没有这些证据，再漂亮的总结也很难支撑长期迭代。
