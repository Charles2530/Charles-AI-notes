# 基础模型系统研究地图

面对扩散模型、VLM、VLA、量化、训练系统、推理系统、世界模型和具身智能这几条快速演进的研究线，很多人最先遇到的问题不是“看不懂某篇论文”，而是“不知道应该先看哪一类问题、这些问题之间如何连接、哪些工作是方法创新、哪些其实是系统整合”。本页不是一篇单一方向综述，而是一张研究地图，帮助你把这些领域放到同一套认知坐标里。

## 1. 先区分四类研究问题

基础模型相关研究大致可以落到四个问题域：

1. **表示问题**：如何学到更好的内部表征。
2. **生成问题**：如何生成图像、视频、文本、动作或未来世界。
3. **决策问题**：如何把表示和生成用于规划、工具使用和控制。
4. **系统问题**：如何把上述能力训练出来、压缩下来、稳定部署出去。

很多论文表面属于某个子领域，实则跨越多个问题域。例如一个 VLA 论文既是多模态表示问题，也是决策问题，同时还依赖训练系统和数据引擎。

## 2. 扩散模型在地图中的位置

**扩散模型最初主要回答生成问题**：如何从复杂高维分布中稳定采样。它的发展线可分为：

1. 训练表示与噪声参数化；
2. 采样器与数值求解器；
3. 蒸馏、DMD、Consistency、Rectified Flow 等快速生成；
4. 多模态、视频、动作与控制扩展。

因此扩散不是孤立的“图像生成技巧”，而是现代生成建模与连续时间学习的重要分支。

## 3. VLM 处在表示与推理的交叉点

**VLM 的问题域横跨**：

1. 跨模态表示对齐；
2. 视觉理解与文本生成；
3. 检索、工具与 agent 系统；
4. 长尾鲁棒性与结构建模。

如果只把 VLM 当作“会看图的 LLM”，就会忽略它对文档、图表、UI、检索和 grounding 的系统要求。

## 4. VLA 连接多模态理解与行动

VLA 站在 VLM 之后，但又不是简单加一个动作头。它把问题转向：

1. 语言如何变成可执行目标；
2. 视觉如何对齐到控制接口；
3. 数据如何从示范、仿真和真实回流中累积；
4. 闭环恢复、Sim2Real 和安全如何建立。

VLA 因此是基础模型与机器人控制的交汇层。

## 5. 世界模型是生成问题与决策问题的桥梁

世界模型既可以看成“生成未来”的模型，也可以看成“为规划提供内部模拟器”的模型。它把研究问题重组为：

1. 怎样压缩并表示环境状态；
2. 怎样预测未来且保留决策相关信息；
3. 怎样用 imagined rollouts 改善规划与数据引擎。

因此世界模型与扩散视频生成有交叉，与 VLA 和具身智能也有交叉。

## 6. 对比学习与自监督表示是底层方法学

对比学习、自蒸馏和非对比方法常被单独成章，但它们更像底层方法学，影响：

1. VLM 的跨模态对齐；
2. 机器人轨迹编码；
3. 检索系统的 embedding；
4. 世界模型的状态表示。

理解这一层，有助于把很多看似不同系统的共性提炼出来。

## 7. 训练系统与推理系统是“能力实现层”

如果把模型研究看成在设计发动机，那么训练和推理系统就是制造厂与公路网络。它们决定：

1. 新能力是否能以合理成本学出来；
2. 是否能在目标硬件上跑起来；
3. 线上服务是否有可接受的延迟、成本和稳定性。

很多论文只给出算法结论，但真正决定可落地性的，往往是系统层细节。

## 8. 量化是连接训练与部署的压缩层

量化不只是部署后处理，也反过来影响：

1. 模型结构设计；
2. 训练中是否考虑低比特友好；
3. KV cache 和长上下文服务；
4. 硬件与 runtime 选择。

因此量化研究最好放在“系统协同”视角下看，而不是孤立看压缩率。

## 9. 如何分阶段阅读

### 9.1 第一阶段：建立主干概念

**建议先理解**：

1. 自回归与扩散两种主要生成范式；
2. 对比学习与自蒸馏的表示学习直觉；
3. 基础训练与推理系统概念；
4. VLM、VLA、世界模型各自解决什么问题。

目标不是记住论文名，而是先建立问题地图。

### 9.2 第二阶段：按问题链阅读

可以按“问题链”而不是按年份看论文。例如：

1. **扩散**：DDPM -> DDIM -> 高阶求解器 -> 蒸馏 / 整流；
2. **表示**：SimCLR/MoCo -> CLIP -> BYOL/DINO -> 多模态检索；
3. **具身**：行为克隆 -> VLA -> Sim2Real -> 闭环恢复；
4. **世界模型**：RSSM/Dreamer -> 视频模型 -> 数据引擎与规划。

这样更容易看出演化动机。

### 9.3 第三阶段：系统化比较

当每条线看过几篇核心工作后，开始比较：

1. 它们各自假设了什么；
2. 最主要瓶颈在哪；
3. 向系统落地时哪一步最难；
4. 哪些研究线正在趋同。

## 10. 一些高价值的比较问题

**阅读时建议持续问自己**：

1. 这篇论文到底优化的是表示、生成、规划还是系统成本？
2. 它解决的瓶颈是“不会做”，还是“做得太慢 / 太贵 / 不够稳”？
3. 它依赖的数据和硬件假设现实吗？
4. 它的收益能否外推到另一个模态或系统中？

这些问题能帮助你从“读论文”过渡到“建立研究判断力”。

## 11. 不同研究角色的阅读路径

### 11.1 偏算法研究者

优先看目标函数、参数化、采样器、表示结构和理论动机。

### 11.2 偏系统研究者

优先看训练并行、推理服务、压缩、缓存、调度与成本建模。

### 11.3 偏产品 / 应用研究者

优先看数据引擎、评测、失败分析、路由和部署案例。

## 12. 为什么要同时读“方法”和“失败”

很多人阅读时只看 SOTA 方法，不看：

1. 失效模式
2. 评测设计
3. 负面结果
4. 工程实现限制

这样很容易形成“方法主义幻觉”。真正好的研究者，既看模型为什么赢，也看它在哪些条件下输。

## 13. 一个示例研究主题的展开方式

假设你想研究“世界模型如何帮助 VLA”。一条有效路径可能是：

1. 先读 Dreamer / RSSM 理解隐状态与 imagined rollout；
2. 再读视频世界模型理解生成式未来表示；
3. 然后读 VLA 数据与控制接口；
4. 最后读 Sim2Real、恢复机制和部署安全。

这样你会看到问题不是“世界模型和 VLA 拼在一起”这么简单，而是表示、规划、控制、数据与评测全部耦合。

## 14. 一个形象比喻

可以把这张研究地图想成一座城市。表示学习像地基和钢筋，生成模型像建筑体量，世界模型与 VLA 像桥梁和交通系统，训练与推理系统像供电供水和道路网络，量化像节能改造，人机协作与安全像城市法规和公共服务。如果你只研究某一栋楼而不看整座城市，很难真正理解哪条路最值得修、哪片区域最容易堵。

## 15. 小结

基础模型研究正在从孤立方法竞赛，逐步转向跨层系统协同。把扩散、VLM、VLA、世界模型、训练、推理、量化和具身智能放在一张统一地图中，有助于你理解：每一条研究线究竟在解决哪类问题，它与其他方向的接口在哪里，以及未来真正高价值的工作，往往发生在这些接口而不是单点之中。
