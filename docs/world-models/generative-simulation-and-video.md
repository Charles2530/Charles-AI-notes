# 生成式模拟与视频世界模型

新一代世界模型越来越像视频生成模型，因为视频天然包含了环境随时间演化的信息。  
如果说传统世界模型更像“在潜空间里写方程”，那视频世界模型更像“直接学会预测未来会看到什么”。

但真正有价值的视频世界模型，并不是单纯把未来画得更漂亮，而是要把“动作会导致什么后果”这件事编码进去。也就是说，它不能只是一个视频续写器，而必须逐渐长成一个可用于规划、风险评估和动作建模的动态系统。

## 1. 统一目标：给定历史与动作，生成未来观测

**最基本的目标可以写成**：

\[
p_\theta(x_{1:T} \mid a_{1:T-1}, c)
\]

其中：

- \(x_{1:T}\) 是未来视频帧；
- \(a_{1:T-1}\) 是动作序列；
- \(c\) 是条件信息，可能是文本、地图、历史观测、任务目标或机器人状态。

更常见的是条件 rollout 形式：

\[
p_\theta(x_{t+1:t+H} \mid x_{\le t}, a_{t:t+H-1}, c)
\]

它描述的是：给定到当前时刻的观察和未来要执行的动作，未来 \(H\) 步会看到什么。

### 1.1 为什么这一定义比普通视频预测更强

普通视频预测很多时候只是在回答：

“接下来最自然的一帧是什么？”

**而世界模型要回答的是**：

“如果我采取动作 \(a\)，会看到什么；如果换成 \(a'\)，未来会怎样变化？”

这要求模型不仅拟合视频分布，还要对动作后果敏感。形式上，需要满足：

\[
p_\theta(x_{t+1:t+H} \mid x_{\le t}, a_{t:t+H-1}, c)
\neq
p_\theta(x_{t+1:t+H} \mid x_{\le t}, a'_{t:t+H-1}, c).
\]

### 1.2 观测不一定只有 RGB 视频

虽然这一页重点讨论视频路线，但在真实系统里，“视频世界模型”的输入常常并不只有 RGB。常见扩展包括：

1. 多视角相机视频；
2. 深度或法向信息；
3. 语义 mask、光流、关键点；
4. 地图切片、几何 token；
5. 机器人 proprioception 或车辆状态。

因此它更准确地说，是“以视频为主观测、但允许多模态条件输入的生成式世界模型”。

## 2. 为什么视频比静态世界模型更直观

视频把很多只在时间里才显现的因素显式暴露出来：

1. 物体连续运动；
2. 遮挡和显现；
3. 接触后的反应；
4. 光照与视角变化；
5. 动作结果反馈；
6. 多主体之间的交互。

对于规划和控制来说，这很重要，因为很多危险并不在当前帧，而在“下一秒会发生什么”。

### 2.1 一个直观例子：仓储机器人推箱子

当前一帧只看到纸箱前面有空位，但视频 rollout 能告诉你：

1. 推 20 厘米后纸箱会不会撞到货架腿；
2. 箱子角度会不会卡在转角；
3. 箱体滑动后夹爪是否还有抓取空间；
4. 推动动作是否会把旁边轻物体带倒。

静态图片很难提供这种因果感，而视频 rollout 天然更接近“内部试错”。

### 2.2 再看一个自动驾驶例子

自动驾驶里，当前一帧可能只看到行人站在人行道边缘。单帧感知告诉你“那里有个人”，但视频世界模型更关心：

1. 他下一秒有没有迈步倾向；
2. 旁边车辆是否会挡住进一步观察；
3. 自车若继续前进，交互分布是否会改变；
4. 如果轻踩刹车，周围交通参与者会如何响应。

这也是为什么视频路线特别适合多主体动态环境。

## 3. 从潜空间动力学到像素级未来生成

若直接在像素空间生成未来帧，目标可以写成自回归形式：

\[
p_\theta(x_{1:T} \mid a_{1:T-1}, c)=\prod_{t=1}^{T} p_\theta(x_t \mid x_{<t}, a_{<t}, c)
\]

但高维像素序列太重，于是很多模型会先压到潜变量：

\[
z_t = E(x_t)
\]

**然后学习潜空间世界模型**：

\[
p_\theta(z_{t+1:t+H} \mid z_{\le t}, a_{t:t+H-1}, c)
\]

**最后再解码**：

\[
\hat{x}_{t+k}=D(z_{t+k})
\]

**这种结构的好处是**：

1. 训练更省；
2. rollout 更长；
3. 更容易和动作、高层条件、地图信息结合；
4. 更方便把世界模型接到规划器，而不是每次完整输出高清视频。

### 3.1 为什么很多系统不会每步都解码视频

对规划来说，真正重要的常常不是每一步像素都看起来完美，而是 latent 中是否保留了：

1. 目标物位置；
2. 接触状态；
3. 速度与姿态；
4. 风险事件；
5. 未来分支差异。

因此实际系统常用“latent rollout + 稀疏解码”策略：大部分时间只在潜空间滚动，只有在需要可视化、打分或人工检查时才解码未来片段。

### 3.2 视频 tokenizer 在这里起什么作用

许多生成式视频世界模型并不是直接处理原始像素，而是先把视频离散化或压缩成 token 序列。这样做的直觉是：

1. 把高维连续像素转成更适合序列建模的单元；
2. 让视频生成能够继承大语言模型或自回归 transformer 的建模方式；
3. 用共享 token 空间统一视频、动作和文本条件。

不过 tokenizer 的好坏非常关键。压得太狠，接触细节和小目标会丢；压得不够，生成成本又会过高。

## 4. 三类常见建模路线

视频世界模型在方法上大致可以分成三条常见路线，它们并不是互斥的，而是经常混合出现。

### 4.1 自回归视频 token 路线

先把视频编码成 token，再按时间或时空顺序预测下一个 token：

\[
p_\theta(v_{1:N} \mid c)=\prod_{i=1}^{N} p_\theta(v_i \mid v_{<i}, c).
\]

**优点是**：

1. 结构统一，容易和语言、动作 token 拼接；
2. 适合做统一序列建模；
3. 很自然地过渡到 `WAM` 风格。

**缺点是**：

1. 序列长，计算成本高；
2. 高频视频细节代价大；
3. rollout 很长时容易积累 token 级误差。

### 4.2 扩散视频路线

把未来视频块当作扩散生成目标，对噪声视频做条件去噪：

\[
X^\tau = \sqrt{\bar{\alpha}_\tau}X + \sqrt{1-\bar{\alpha}_\tau}\epsilon
\]

\[
\mathcal{L}_{\epsilon} =
\mathbb{E}\|\epsilon - \epsilon_\theta(X^\tau, x_{\le t}, a_{t:t+H-1}, c, \tau)\|_2^2.
\]

**优点是**：

1. 视觉质量通常更强；
2. 更适合表达多模态未来；
3. 对复杂运动和纹理生成能力较好。

**缺点是**：

1. 推理成本高；
2. 直接用于高频在线控制很难；
3. 看起来逼真不代表动力学可用于规划。

### 4.3 潜空间动力学加视频解码路线

先学一个可 rollout 的 latent dynamics，再把预测 latent 解码成视频或结构化观测。这条路线更接近经典 `WM`，只是把“观测头”升级成强视频解码器。

**优点是**：

1. 更适合规划；
2. rollout 成本低；
3. 容易加入 reward、risk、termination 等辅助头。

**缺点是**：

1. 解码出的视觉质量不一定最好；
2. latent 若设计不当，可能保不住对控制关键的细节；
3. 很依赖隐状态是否真的学到长期因果结构。

### 4.4 三条路线各自适合什么

**一个粗略判断是**：

1. 如果主要目标是高质量未来视频与多模态想象，扩散路线通常更有吸引力；
2. 如果主要目标是统一观测 token、动作 token 和文本 token，自回归路线更自然；
3. 如果主要目标是规划、评分和 rollout 效率，潜空间动力学路线更实用。

真实系统里最常见的并不是纯粹某一种，而是把三者拼起来：例如用视频 tokenizer 压缩，再用自回归或扩散建模未来 latent，最后只在必要时解码到像素空间。

## 5. 常见训练目标

### 5.1 重建损失

若直接预测像素，可写作：

\[
\mathcal{L}_{\text{rec}}=\sum_{t=1}^{T}\|x_t-\hat{x}_t\|_1
\]

或使用感知损失、对数似然、patch-level 误差等。

### 5.2 潜变量建模

若引入潜变量 \(z_t\)，则常见目标为：

\[
\mathcal{L} =
\sum_t \mathbb{E}_{q(z_t)}[-\log p_\theta(x_t \mid z_t)]
+
\beta \sum_t D_{\text{KL}}(q(z_t)\|p_\theta(z_t \mid z_{<t}, a_{<t}, c))
\]

它要求 latent 既能重建当前观测，又能按动作条件稳定向前滚动。

### 5.3 动作一致性目标

如果模型真要服务控制，就不能只让未来“看起来对”，还要让动作差异体现在未来差异里。可以引入动作敏感一致性目标，概念上写成：

\[
\mathcal{L}_{\text{act}}
=
- d\Big(
\hat{x}^{(a)}_{t+1:t+H},
\hat{x}^{(a')}_{t+1:t+H}
\Big)
\]

其中 \(a \neq a'\)，要求模型对不同动作给出足够可区分的未来。这个写法只是直觉表达，实际实现可能是 contrastive loss、ranking loss 或结构化 supervision。

### 5.4 任务相关辅助头

视频世界模型若要真正在机器人和自动驾驶里落地，往往还会附带多个辅助预测头：

1. reward / success 头；
2. risk / collision 头；
3. contact 或 event 头；
4. segmentation / depth 头；
5. map-alignment 或 lane occupancy 头。

这类目标的作用是防止模型只学到“看起来像”，却没把任务关键信息编码进 latent。

## 6. 一个重要区别：逼真不等于可规划

视频世界模型最大的陷阱，就是它很容易“看起来真”，却未必“对决策有用”。

假设模型生成的未来视频很清晰，阴影和纹理都很真实，但物体实际位移方向错了。  
从生成角度，它可能得分很高；从规划角度，它等于误导控制器。

所以对世界模型来说，真正关键的不只是：

1. 帧看上去是否自然；
2. 纹理是否逼真；
3. 视频是否连贯。

**更重要的是**：

1. 物体位置对不对；
2. 速度和接触结果对不对；
3. 动作变化是否引起正确未来差异；
4. 多模态关键分支是否被保留；
5. 风险事件是否被“美化”掉。

## 7. 动作条件生成为什么重要

很多视频模型只会预测“自然下一帧”，但世界模型需要回答更强的问题：

“如果我做动作 \(a\)，未来会怎样；如果换动作 \(a'\)，未来又怎样？”

### 7.1 动作条件可以怎么注入

动作并不只是简单拼接到输入后面，常见注入方式包括：

1. 把动作编码成和视频 token 同级的序列 token；
2. 把动作映射成 cross-attention 条件；
3. 在 latent transition 中直接作为控制变量；
4. 把高层 goal 先翻译成未来动作草图，再条件生成视频。

不同注入方式的差别，决定了模型更像视频预测器、还是更像统一 action-conditioned generator。

### 7.2 连续动作和离散动作并不一样

机器人中的关节控制、末端位姿和抓手开合通常是连续变量；驾驶中的轨迹点、曲率和纵向加速度也经常连续。为了让生成模型更稳定，人们常会做：

1. 动作离散化；
2. 分桶量化；
3. 低维 skill token 化；
4. 高层动作与低层控制分离。

这样做会影响模型的可控性。如果量化过粗，动作差异在视频里不明显；如果量化太细，训练又很难稳定。

### 7.3 一个自动驾驶换道例子

**给定当前高速路画面**：

1. 若动作是“保持车道”，未来可能逐渐接近前车；
2. 若动作是“左变道”，未来画面应显示车身向左平移、左侧车道车辆距离变化；
3. 若动作是“减速观察”，未来分布则应更多体现后车逼近与空隙演化。

如果模型对这三组动作生成的未来几乎一样，那它就不是合格的世界模型，只是一个会“续写视频”的生成器。

## 8. rollout 到底怎么在系统里用

视频世界模型的价值，不在于每次都完整生成长视频，而在于它能支持几种关键使用方式。

### 8.1 候选动作前瞻

给出若干候选动作或轨迹，让模型分别 rollout 若干未来，再比较：

1. 哪个更安全；
2. 哪个更容易成功；
3. 哪个更有利于后续子任务；
4. 哪个更接近任务目标。

### 8.2 失败回放与反事实

给定一段真实失败轨迹，从失败前若干步开始，替换不同动作进行 counterfactual rollout，帮助回答：

1. 问题从哪一步开始不可逆；
2. 当时是否存在更安全动作；
3. 失败来自观测、动作还是动力学偏差。

### 8.3 imagined data

在高置信区域生成 imagined future，可用于：

1. value warmup；
2. curriculum 样本扩增；
3. 风险场景补全；
4. 人工筛查候选片段。

不过 imagined data 必须受控使用，因为 model bias 会被直接带入下游策略。

### 8.4 视频解码不一定发生在在线控制主路径

在很多系统里，在线主路径只用 latent rollout 和风险/价值头，完整视频解码只用于：

1. 离线诊断；
2. 可解释性展示；
3. 数据筛查；
4. 人工审核；
5. 少量关键时刻的辅助判断。

这样更符合真实延迟预算。

## 9. 从视频世界模型到 `VAM`

视频世界模型继续往机器人或驾驶控制方向走时，常会自然长成 `VAM`（Video-Action Model）风格。  
这条路线的核心变化不是“多了个动作头”这么简单，而是：

1. 视频 latent 不再只服务未来生成；
2. 它开始直接服务动作生成或动作解码；
3. 视频和动作之间开始共享同一套时空表示；
4. 视频预训练能力开始真正转移到控制任务上。

**可把这种联合目标粗略写成**：

$$
\mathcal{L}_{\text{VAM}}
=
\mathcal{L}_{\text{video}}
+ \lambda \mathcal{L}_{\text{action}}
+ \eta \mathcal{L}_{\text{joint-latent}}.
$$

### 9.1 为什么很多团队会走向 `VAM`

因为单纯的 `VLA` 常依赖静态图像或短轨迹数据，很难天然学到物理因果和长期时序。  
**而视频先验天然包含**：

1. 物体运动；
2. 接触结果；
3. 遮挡变化；
4. 手部与工具交互；
5. 动作导致的视觉后果。

这也是 `mimic-video` 这类工作强调“用互联网规模视频模型补足静态视觉先验不足”的原因。[mimic-video](https://arxiv.org/abs/2512.15692)

### 9.2 `VAM` 常见的两种做法

#### 先视频，后动作

先训练或利用强视频模型，再从视频 latent 中解码动作。  
这条路优点是复用强预训练模型容易，缺点是视频 latent 未必天然适合动作解码。

#### 视频动作联合优化

让视频预测和动作预测在同一 latent 里共同训练。  
像 `Unified Video Action Model` 就属于这种代表性路线：它强调 joint latent，但推理时又把 video decoding 和 action decoding 解耦，以保动作推理速度。[Unified Video Action Model](https://arxiv.org/abs/2503.00200)

### 9.3 `VAM` 真正要验证的不是“能生成视频”

`VAM` 的关键问题是：

1. 视频先验是否真的提高了动作质量；
2. 视频 latent 是否保留了控制最需要的信息；
3. 不解码视频时，动作分支是否仍然受益；
4. 训练成本是否值得换来更好的泛化。

很多系统最后的有效形态并不是“天天生成视频再控制”，而是“训练时借视频世界建模学到更好的动作表征，部署时主要走动作分支”。

## 10. 从视频世界模型到 `WAM`

另一条更偏统一生成的路线，会把未来世界和未来动作一起当作模型输出，也就是更接近 `WAM`（World-Action Model）。

若写成序列建模，它更像：

$$
p(x_{t+1:t+H}, a_{t:t+H-1} \mid x_{\le t}, g).
$$

**这里的重点是**：动作不再只是视频模型的外部条件，而是和未来世界一起被模型“续写”出来。

### 10.1 为什么视频世界模型会自然走到 `WAM`

**因为一旦你已经能做**：

1. 条件未来视频生成；
2. 动作条件控制；
3. 多步 rollout；
4. 高层目标条件化；

**下一步最自然的问题就是**：

“那我能不能把动作本身也纳入统一生成，让动作和未来相互约束？”

这就很容易长成 `WAM`。

### 10.2 `WAM` 比单纯视频条件生成更进一步的地方

在单纯视频条件生成里，动作通常是外部输入；  
在 `WAM` 里，动作和世界未来一起进入同一联合分布。这样做的潜在好处是：

1. 动作和结果之间耦合更强；
2. 统一序列更适合做 planning-as-generation；
3. 容易和语言目标、地图 token、状态 token 放在一起建模；
4. 可以用同一模型支持行为生成、未来想象和候选筛选。

### 10.3 当前代表性的趋势

`WorldVLA` 和 `DriveDreamer-Policy` 都体现了这种趋势：  
前者把 action model 与 world model 放进同一自回归框架，后者则更进一步强调 geometry-grounded 的 world-action modeling，用统一框架同时生成深度、未来视频和动作规划。[WorldVLA](https://arxiv.org/abs/2506.21539) [DriveDreamer-Policy](https://arxiv.org/abs/2604.01765)

## 11. 三类常见用途

### 11.1 规划前瞻

先生成几条未来，再比较哪条更安全或更高收益。

### 11.2 数据增强和离线模拟

在真实数据不足时，世界模型可生成近似未来，用于策略 warmup、异常场景回放、near-miss 挖掘或 counterfactual 分析。

### 11.3 表征学习

即使不直接用视频预测结果做控制，学习未来生成也会迫使模型掌握：

1. 物体恒常性；
2. 遮挡关系；
3. 动作后果；
4. 交互结构；
5. 长时动态摘要。

这些表征对下游策略学习有帮助。

### 11.4 动作先验学习

这是近两年视频路线越来越重要的新用途。  
**也就是让模型通过视频先验学到**：

1. 哪些动作序列更自然；
2. 哪些接触过程更合理；
3. 什么样的视频 latent 更容易翻译成控制动作；
4. 哪些未来分支值得优先探索。

这正是 `VAM` 路线特别强调的收益。

## 12. 四个更具体的例子

### 12.1 家庭机器人收拾桌面

如果世界模型能生成“把杯子向左推 10 厘米后的未来画面”，策略就能判断：

1. 杯子会不会碰到盘子；
2. 桌沿是否太近；
3. 推完后手爪是否还有抓取空间；
4. 这个动作是否会妨碍后续放进盒子的姿态。

### 12.2 自动驾驶路口决策

当前看到行人站在人行道边缘。  
短期视频生成若能正确反映“行人下一秒可能迈步进入道路”，就能显著帮助风险评估。

### 12.3 无人机穿林

森林中树枝密集、视角变化快。  
视频世界模型若能提前生成未来 1 到 2 秒视野，规划器就更容易判断当前速度下是否会擦到右前方树枝，以及减速或偏航是否更安全。

### 12.4 工厂机械臂插接

插头对准插座前的最后几毫米，位置误差很小但代价很大。  
若世界模型能预测“继续向前 5mm 是否会偏斜卡住”，它对精细装配就很有价值。不过这类场景也提醒我们：视频世界模型并不能天然替代精密物理建模。

## 13. 评测视频世界模型时应看什么

除了常见的视频生成指标，还应看：

1. 物体轨迹误差；
2. 动作敏感性；
3. 长时 rollout 漂移；
4. 规划后策略成功率；
5. 风险事件召回；
6. 不确定性和真实误差的相关性。

可以定义一个 rollout 误差：

\[
\mathcal{E}_{\text{rollout}}(H)=\sum_{k=1}^{H}\|g(x_{t+k})-g(\hat{x}_{t+k})\|
\]

其中 \(g(\cdot)\) 提取任务相关状态，例如物体位置、车道中心偏差、碰撞标志。  
这通常比纯像素误差更有意义。

### 13.1 如果方法开始声称自己是 `VAM` 或 `WAM`，还要多看两件事

1. 动作质量是否真的比直接 policy learning 更强；
2. 视频分支或世界分支是否真正帮助了动作，而不是只增加了训练负担。

很多统一模型论文的关键，不在于它能不能生成视频，而在于联合建模是否真的带来了控制收益。

## 14. 主要风险

### 14.1 视频逼真不代表动力学正确

这是最大陷阱。一个模型可以把阴影、纹理和流畅感做得很好，却在关键位置关系上完全错误。

### 14.2 长时预测误差快速累积

滚动几十帧后，哪怕前面每步只差一点，最后也可能完全跑偏。长时 rollout 的稳定性通常远比单步画面质量更重要。

### 14.3 稀有危险事件学不到

训练数据里很少有真正危险的 near-miss 场景，于是模型在最关键的时刻最不可靠。

### 14.4 模型会“美化”未来

生成模型天生倾向于输出平均意义上平滑、自然的画面。  
可现实世界里的重要事件往往恰恰是不平滑、不常见的那一小撮异常。如果模型把异常分支洗成“看起来更正常”的未来，就会直接误导规划。

### 14.5 成本过高导致无法进入主链路

有些视频世界模型离线展示效果很好，但编码、采样、解码总时延太高，根本无法进入真实决策主路径。工程上这类模型仍可能有价值，但通常只能做离线分析、数据筛查或低频辅助模块。

## 15. 工程判断

**若你做的是**：

1. 视觉预测；
2. 风险前瞻；
3. 高成本策略试错；
4. 需要从海量视频吸收时空先验；

视频世界模型非常值得投入。  
但如果你要的是严格物理仿真，例如精密接触动力学，它通常还不能完全替代高保真物理引擎。

### 15.1 一个更细的路线判断

**若你主要关心**：

1. imagined planning；
2. 风险前瞻；
3. 闭环控制中的未来评估；

优先看偏 `WM / WAM` 的路线。

**若你主要关心**：

1. 视频先验迁移；
2. 动作泛化；
3. 利用互联网视频改善机器人控制；

优先看偏 `VAM` 的路线。

**若你主要关心**：

1. 把动作和世界统一成单个生成问题；
2. 统一视频、语言、动作和目标；
3. 让 planning 更像 sequence generation；

则更应关注偏 `WAM` 的统一建模路线。

## 16. 总结

视频世界模型的真正意义，不是把未来生成得像电影，而是把“行动后会看到什么”变成可学习、可采样、可比较的对象。  
一旦这件事做得足够准，智能体就不再只是被动响应当前帧，而能真正基于未来后果进行选择。

如果再往前走一步，那么视频模型就不只是服务于“看见未来”，而会开始服务于“生成动作”“联合规划”和“构建动作先验”。这也正是为什么今天的视频世界模型，正在不断向 `VAM` 与 `WAM` 汇合。

## 17. 视频 tokenizer 与 latent 表示为什么是关键前置模块

很多视频世界模型真正的瓶颈，并不是 transformer 或 diffusion 主干，而是“视频到底先被压成什么表示”。这是因为世界模型最终既要：

1. 吃下大规模时空观测；
2. 保留对控制有用的局部细节；
3. 允许高效 rollout；
4. 和动作、语言、地图等其他模态对齐。

### 17.1 离散 token 路线

**优点**：

1. 更容易继承语言模型式序列建模；
2. 容易和文本、动作 token 拼接；
3. 便于做统一自回归建模。

**代价**：

1. 序列太长；
2. 高频细节压缩困难；
3. 接触和微小几何误差容易丢失。

### 17.2 连续 latent 路线

**优点**：

1. rollout 成本更低；
2. 更适合和经典 latent world model 对接；
3. 更容易接 reward、risk、termination 头。

**代价**：

1. 可解释性弱；
2. 需要额外验证 latent 是否保留了决策关键信息；
3. 联合语言和动作时接口设计更难。

### 17.3 为什么很多系统采用“压缩 + 稀疏解码”

在规划和控制场景里，系统常常只需要：

1. 高频 rollout；
2. 低频可视化；
3. 在关键时刻解码确认未来分支。

因此很多视频世界模型不会每步都重建清晰视频，而是：

1. 大多数时间在 latent 中滚动；
2. 只在候选动作比较、人工检查或风险解释时再解码部分未来。

## 18. 视频世界模型和 `WM / WAM / VAM` 的关系

视频路线并不等于 `VAM`，但它确实是 `VAM` 非常自然的技术底盘。一个粗略的理解是：

1. 偏经典 `WM` 的视频路线，更强调“给定动作，未来视频怎么变”；
2. 偏 `WAM` 的路线，会把未来视频和未来动作一起联合建模；
3. 偏 `VAM` 的路线，则更明确地把视频建模先验直接当成动作泛化和控制学习的基础。

### 18.1 为什么这个区分在工程上有用

**因为三者的投入重点并不一样**：

1. 若目标是想象未来与做风险评估，重点是 rollout 质量和动作敏感性；
2. 若目标是统一动作和世界生成，重点是 joint modeling 接口；
3. 若目标是利用大规模视频先验帮助机器人，重点是视频表示与动作迁移。

## 19. 闭环控制为什么比离线视频指标更难

很多视频生成指标在世界模型场景里参考价值有限。原因是：

1. 画面看起来更真，不代表动作后果更准；
2. 平均视频质量高，不代表 near-miss 场景建模更好；
3. 离线 rollout 漂一点点，闭环控制里会被放大。

### 19.1 为什么 closed-loop 验证是必须的

**真正要回答的问题是**：

1. 用这个模型做动作选择，成功率有没有提升；
2. 风险事件是否更早被识别；
3. 长时任务是否更稳；
4. 数据引擎是否更容易找到高价值错误样本。

如果这些问题没有改善，再好看的视频也只是展示能力。

## 20. 视频世界模型的数据问题比普通视频生成更难

因为这里的“好数据”不仅是清晰和多样，更要有：

1. 可用动作标签；
2. 任务相关事件；
3. 风险和 near-miss 片段；
4. 多视角或附加状态；
5. 时间对齐质量。

### 20.1 为什么互联网视频不够

**互联网视频可以提供**：

1. 运动先验；
2. 物体交互先验；
3. 长时场景变化先验。

**但它通常缺**：

1. 精确动作标签；
2. 可执行控制接口；
3. 任务成功与失败标注；
4. 高风险边界数据。

因此互联网视频更像预训练底座，而不是直接可用的闭环控制数据。

## 21. 自动驾驶和机器人为什么都爱用视频路线，但目标不同

### 21.1 自动驾驶

**更关心**：

1. 多主体互动；
2. 长时风险演化；
3. near-miss 和稀有事件；
4. 地图与几何约束。

### 21.2 机器人

**更关心**：

1. 接触与遮挡；
2. 任务步骤切换；
3. 精细操作前的局部未来；
4. 动作可执行性和恢复。

这意味着二者虽然都用“视频世界模型”这个名字，但对 latent、解码、动作接口和评测的要求并不完全一样。

## 22. 视频世界模型在数据引擎里的作用

即使模型还不足以直接主导闭环控制，它也可能非常适合做：

1. near-miss 发现；
2. 反事实生成；
3. 数据重采样优先级排序；
4. 合成新场景和新视角；
5. 失败恢复候选动作筛选。

### 22.1 为什么这很现实

**很多团队最终会采用**：

1. 低频使用视频世界模型做离线分析和数据引擎；
2. 高频在线控制仍使用更轻的 latent dynamics 或传统控制器；
3. 两者通过风险评分和样本回流连接。

这通常比“直接让重视频模型上主控制环”更可落地。

## 23. 失效模式再细分

### 23.1 视觉上对、动力学上错

物体位置、接触时机或速度变化稍错一点，就可能导致规划彻底跑偏。

### 23.2 学会了背景，不会学动作后果

模型能很好续写场景纹理，却没有真正建立动作和结果之间的因果联系。

### 23.3 过于依赖数据集中常见未来

面对少见但关键的危险分支，模型会自动“美化”为常见平滑未来。

### 23.4 在线开销不可接受

编码、采样、解码的总时延太大，导致模型只能做离线辅助，而进不了控制主链。

## 24. 设计建议

1. 先明确目标是视频预测、规划辅助、动作联合建模还是数据引擎；
2. 视频 tokenizer 设计时优先验证控制关键信息是否保留；
3. 离线指标之外，必须做 closed-loop 或至少 task-aware rollout 验证；
4. 把 near-miss、风险事件和动作敏感性单独拉成评测桶；
5. 若在线成本过高，优先考虑把视频世界模型放入低频分析或数据引擎环节。

## 25. 小结

生成式模拟与视频世界模型，不应被理解成“把视频生成技术搬进控制领域”这么简单。真正关键的是：如何把视频中的时空先验、动作后果和风险分支压成一个既能想象未来、又能服务决策的中间层。谁能把这件事做对，谁就更有机会把 `WM`、`WAM`、`VAM` 这些路线真正连到机器人、驾驶和具身系统的闭环里。

## 26. 视频 tokenizer 是这条路线最关键的地基之一

很多生成式模拟系统的上限，早在 tokenizer 这一层就被决定了。**如果 tokenizer 丢掉了控制相关细节，后面世界模型再强，也只是在一个已经受损的表征空间里做预测。**

### 26.1 视频 tokenizer 最该保住什么

1. **运动边界和速度变化**；
2. **接触、遮挡、滑移、碰撞等动态事件**；
3. **与任务相关的小物体与长尾对象**；
4. **时间连续性**；
5. **多视角或多传感器间的对应关系**。

### 26.2 为什么“看起来清晰”不够

一个能生成清晰视频的 tokenizer，不一定保留了：

1. **机械臂夹爪与物体微小相对位移**；
2. **车辆与路沿、人行横道、侧向来车的精细关系**；
3. **风险边缘事件的临界状态**。

所以 tokenizer 评测不能只看 FVD、PSNR 或重建观感，还要看**控制相关可读性**。

## 27. 生成式模拟的数据引擎通常比模型本身更重

要把视频世界模型做起来，数据引擎常常比核心模型更先成为瓶颈。

### 27.1 为什么

1. **视频数据量大，清洗和切片成本高**；
2. **动作、语言、传感器、地图等旁路信号需要严格对齐**；
3. **高价值样本往往是 near-miss、罕见操作、失败恢复，不是平均场景**；
4. **训练和评测需要不同粒度的 clip、episode 与事件标签**。

### 27.2 一条更现实的管线

1. **原始多模态日志入湖**；
2. **事件切分和对齐**；
3. **关键片段挖掘**，如碰撞前数秒、接触前后、异常恢复片段；
4. **面向训练的 tokenization 与 pack**；
5. **面向评测和回放的高保真存档**。

这意味着：**生成式模拟不是只有模型工程，它先是数据工程。**

## 28. Diffusion、AR、Latent Rollout 三种主线怎么选

视频世界模型常在三类生成范式间权衡：

1. **diffusion**；
2. **autoregressive token rollout**；
3. **latent dynamics rollout**。

### 28.1 Diffusion 的优势与代价

**优势**：

1. **画质和多峰未来表达通常更强**；
2. **对高维视觉建模更自然**；
3. **容易表达复杂随机性**。

**代价**：

1. **采样成本高**；
2. **在线控制难直接承受**；
3. **动作敏感性不一定天然强**。

### 28.2 AR rollout 的优势与代价

**优势**：

1. **统一 token 接口，便于把观测和动作放进同一序列**；
2. **易于做 next-token 式建模与大规模预训练迁移**；
3. **适合统一多模态事件序列**。

**代价**：

1. **长 horizon 暴露误差积累**；
2. **时空结构若编码不好，会变成长序列负担**；
3. **高保真视频细节不一定最优**。

### 28.3 Latent rollout 的优势与代价

**优势**：

1. **计算更省**；
2. **更适合规划和控制接口**；
3. **可把视频只作为训练信号，而不必每次完整解码**。

**代价**：

1. **可解释性较弱**；
2. **latent 是否保留任务关键物理结构更难验证**；
3. **若想反向生成视频回放，仍需额外解码器**。

### 28.4 一个常见落地策略

很多系统最终不是三选一，而是：

1. **用视频 tokenizer 学视觉压缩**；
2. **在 latent 空间做主 rollout**；
3. **只在需要回放、解释或生成风险分支时解码视频**。

## 29. 机器人和驾驶里的视频世界模型关注点不同

### 29.1 机器人更关注

1. **手眼协调和接触细节**；
2. **小范围空间内的高精度动态变化**；
3. **动作和物体状态之间的短程高灵敏耦合**；
4. **低延迟闭环预算**。

### 29.2 自动驾驶更关注

1. **多实体交互与长时场景演化**；
2. **地图、车道拓扑与交通规则的先验**；
3. **风险分支与 near-miss 预测**；
4. **多摄像头、多模态融合与规模化评测**。

### 29.3 这影响什么

1. **tokenizer 粒度**；
2. **动作接口**；
3. **horizon 长度**；
4. **评测重点**；
5. **是否需要显式场景结构，如 occupancy、轨迹或实例级对象状态**。

## 30. 显式场景结构和纯视频生成不是对立面

很多讨论会把“视频世界模型”与“结构化场景模型”对立起来，其实更现实的趋势是**两者互补**。

### 30.1 显式结构有什么价值

1. **规划器更容易消费**；
2. **风险规则更容易接入**；
3. **可解释性更强**；
4. **更适合长尾事件统计**。

### 30.2 纯视频生成有什么价值

1. **保留细粒度视觉细节**；
2. **能覆盖结构化标签难表达的复杂场景**；
3. **适合做高保真反事实回放**。

### 30.3 更有前景的组合

1. **共享 backbone / tokenizer**；
2. **同时预测视频 latent 和结构化场景头**；
3. **规划主要用结构化头，验证和解释用视频回放**。

## 31. 控制环里最重要的是“生成什么频率、在哪个环节用”

视频世界模型常被质疑“太慢”，这其实是在问：**它到底放在哪个控制环里**。

### 31.1 高频主环

若模型要进高频主环，要求最苛刻：

1. **latent rollout 足够快**；
2. **动作敏感性强**；
3. **不确定性能及时给出**；
4. **推理预算可预测**。

### 31.2 中频规划环

如果放在较低频的规划或 proposal 环，模型可以更重一些，重点变成：

1. **候选未来分支质量**；
2. **风险排序能力**；
3. **与快速执行器的接口是否清楚**。

### 31.3 低频分析环

若用于离线回放、事故分析、数据回流或评测，视频生成质量和多分支覆盖度会比实时性更重要。

## 32. Counterfactual Simulation 的价值常高于平均 rollout 质量

对很多真实系统来说，生成式模拟最值钱的不是“平均未来预测更准”，而是它能回答：

1. **如果当时方向盘多打 5 度会怎样**；
2. **如果机械臂晚 0.3 秒闭合夹爪会怎样**；
3. **如果障碍物突然进入视野，当前策略有没有回退路径**。

### 32.1 这为什么比平均指标更关键

因为真实闭环系统最关心的往往不是平均日常样本，而是**决策边界附近的罕见样本**。视频世界模型如果不能支持这些反事实分支，它对安全和策略改进的价值会大打折扣。

## 33. 风险建模需要被显式地放进视频世界模型

很多系统把风险评估留给下游模块，但随着视频世界模型变强，更合理的做法往往是：

1. **在模型中显式预测风险头或 hazard score**；
2. **把 near-miss、违规、碰撞前兆做成专项训练桶**；
3. **让生成未来不仅有“会发生什么”，还有“多危险”**。

### 33.1 为什么不能完全依赖视频回放后人工判断

1. **人工判断扩展性差**；
2. **很多线上回路需要自动回退**；
3. **风险值若不能直接读出，就难以进入规划成本函数或调度逻辑**。

## 34. 一份生成式模拟系统的工程清单

如果一个团队要把视频世界模型从研究原型推到工程链路，建议至少回答：

1. **视频 tokenizer 是否保留控制关键细节**；
2. **动作条件是否真正进入生成主干**；
3. **训练数据是否覆盖 near-miss 和失败恢复**；
4. **模型会进入高频、中频还是低频环**；
5. **是否支持 counterfactual rollout**；
6. **风险与不确定性是否有显式读出**；
7. **视频回放与结构化评测是否能互相校验**。

## 35. 小结

生成式模拟与视频世界模型的真正难点，不只是把视频生成得更像，而是把**视频先验、动作条件、风险分支、结构化状态和控制接口**编织成一个可用系统。谁能在这几层之间做出清晰分工，谁才更有可能让视频世界模型从“会想象”走向“能帮助决策”。
