# 模拟器、反事实生成与合成 Rollout 数据

世界模型一旦不仅用于“预测未来”，而开始用于“指导下一步收什么数据、在哪些场景做更强验证、如何构造更高价值训练样本”，它就会自然走向另一个重要方向：模拟器、反事实生成和合成 rollout 数据。这条路线的核心不在于“用模型伪造更多样本”这么简单，而在于如何让 imagined data 真正服务于：

1. 长尾风险覆盖；
2. 失败恢复训练；
3. planner warmup；
4. 策略改进；
5. 验证与 stress testing；
6. 数据引擎闭环。

这页重点讨论的不是单纯生成更多视频，而是：

1. 世界模型如何与经典模拟器协同；
2. 反事实数据为什么在机器人和驾驶里特别有价值；
3. imagined rollout 数据如何安全地进入训练；
4. 为什么合成数据门禁和验证，往往比生成本身更重要。

## 1. 为什么世界模型天然会走向“模拟器化”

**一旦模型能够学习**：

\[
p(o_{t+1:t+H} \mid o_{\le t}, a_{t:t+H-1}, g),
\]

它就不只是一个预测器，而开始接近一个可条件查询的环境模拟器。  
**你可以问它**：

1. 如果现在向左绕开，会发生什么；
2. 如果再多等一秒，会发生什么；
3. 如果换一种抓取姿态，成功率会怎样；
4. 如果面对同样目标但障碍位置不同，又会怎样。

**这正是模拟器的核心价值**：提供安全、廉价、可重复的“先试一遍”能力。

## 2. 世界模型模拟器与经典物理模拟器不是替代关系

### 2.1 经典模拟器擅长什么

1. 明确几何和动力学；
2. 可控参数化；
3. 严格约束和可重复实验；
4. 对规则变化有清晰解释。

### 2.2 世界模型模拟器擅长什么

1. 从真实数据中学习复杂环境先验；
2. 吸收现实中的感知噪声、遮挡和多主体互动；
3. 建模难以手工写出的策略与行为分布；
4. 更自然地用于反事实和数据引擎。

### 2.3 为什么两者应混合

在很多现实系统里，更有效的方案并不是二选一，而是：

1. 经典模拟器提供结构约束和可控场景；
2. 世界模型提供真实感、行为先验和难显式建模的分布；
3. 两者共同服务训练、验证和数据生成。

## 3. 什么是反事实生成

**反事实生成的核心问题是**：

**如果只改变某个因素，其余条件尽量保持不变，未来会怎么改？**

例如：

1. 同样的桌面场景，把杯子位置偏 5 厘米；
2. 同样的路口，把行人启动时间提前 0.7 秒；
3. 同样的网页流程，把一个按钮位置或文本改掉；
4. 同样的工具调用，把一条约束改成不同数值。

### 3.1 为什么反事实这么值钱

因为真实数据最稀缺的部分，往往恰恰是：

1. 接近失败边界但又没完全失败的样本；
2. 差一点就成功或差一点就出事故的样本；
3. 人工示范很难系统覆盖的局部变化。

反事实让系统能围绕高价值边界做更密集的学习。

## 4. 反事实数据对机器人特别有用

在机器人里，最有价值的样本往往不是普通成功，而是：

1. 抓取点稍偏时会不会滑；
2. 推动物体多一点会不会碰倒旁边杯子；
3. 抽屉开合角度不同会不会卡住；
4. 放置位置偏一点会不会掉落。

### 4.1 为什么真实采集很贵

1. 需要真实试错；
2. 容易损坏物体；
3. 操作者时间昂贵；
4. 很难系统覆盖所有边界条件。

所以世界模型一旦能生成可信的反事实 rollout，它就能极大提高边界数据密度。

## 5. 自动驾驶为什么也离不开反事实

驾驶场景里，很多最关键的数据来自 near-miss：

1. 行人是否迈步；
2. 旁车是否礼让；
3. 施工锥桶是否稍微偏移；
4. 自车若早刹还是晚刹，风险如何变化。

### 5.1 为什么真实世界很难直接采这些数据

因为很多危险边界不可能通过真实主动试错来收集。  
于是反事实模拟就变得极其关键，它允许系统围绕同一真实片段构造：

1. 不同动作条件；
2. 不同交通参与者反应；
3. 不同环境条件；
4. 不同风险分支。

## 6. 合成 rollout 数据的几种主要用途

### 6.1 Planner warmup

在真实交互很贵时，先用 imagined rollout 让 planner 或 value 头有个初始形状。

### 6.2 Risk head 强化

利用高风险反事实样本，增加对 near-miss 与事故边界的识别能力。

### 6.3 恢复策略训练

合成“接近失败但仍可救回”的轨迹，教系统如何恢复。

### 6.4 数据平衡

真实数据里某些边界极少，合成 rollout 可用于补全覆盖。

### 6.5 验证和 stress testing

不一定进入训练，也可作为专项评测集使用。

## 7. imagined data 最危险的地方在哪里

最危险的不是它“不真实”，而是它可能在你最在意的地方不真实。  
**常见风险包括**：

1. 平均上看着合理，边界上完全错误；
2. 视频很真，但动力学错；
3. 风险事件被平滑化；
4. 模型偏差被重复蒸馏；
5. planner 学会利用模型漏洞。

### 7.1 为什么门禁比生成更重要

如果没有门禁，系统只会越来越自信地学习自己的幻觉。  
因此 imagined data pipeline 的核心，不是“怎么生更多”，而是：

1. 哪些能进训练；
2. 哪些只作为弱监督；
3. 哪些只能进验证集；
4. 哪些必须经过人工或经典模拟器复核。

## 8. imagined data 的门禁设计

一个更稳妥的 imagined data 流程通常会分层：

1. `candidate pool`：世界模型生成的候选反事实或 rollout；
2. `consistency check`：与几何、物理、工具规则或程序约束做一致性检查；
3. `disagreement filter`：看 ensemble 或多模型是否意见分歧过大；
4. `human / simulator gate`：高风险样本交由人工或经典模拟器复核；
5. `training export`：按用途送入不同训练池。

### 8.1 为什么要分用途导出

因为 imagined data 不一定适合同样的训练目标：

1. 高置信 rollout 可作为辅助监督；
2. 中置信 rollout 可用于 risk ranking；
3. 高分歧样本可优先送人工；
4. 有明确漏洞风险的样本只适合作验证，不适合直接喂主模型。

## 9. 模拟器与世界模型的三种协同方式

### 9.1 Simulator-to-WM

先用经典模拟器生成结构化数据，再用世界模型学习更强表征和更高吞吐 rollout。

### 9.2 WM-to-Simulator

用世界模型发现高价值边界场景，再把这些场景参数送入经典模拟器做高保真验证。

### 9.3 Hybrid-in-the-loop

世界模型和经典模拟器共同参与闭环：

1. 世界模型负责快速筛选和生成候选；
2. 经典模拟器负责高保真复核关键分支；
3. 数据引擎负责决定哪些样本值得保留。

## 10. 这条路线如何连接 `WM / WAM / VAM`

### 10.1 `WM`

**更自然地服务于**：

1. imagined planning；
2. 反事实未来预测；
3. 风险场景挖掘；
4. 环境状态覆盖扩展。

### 10.2 `WAM`

**更自然地服务于**：

1. 联合生成动作与世界后果；
2. 用动作条件合成恢复轨迹；
3. 围绕任务目标生成多种行为分支。

### 10.3 `VAM`

**更自然地服务于**：

1. 视频反事实；
2. 视角和场景变化增强；
3. 通过视频先验扩展动作数据；
4. 用视频未来帮助构造更真实的合成样本。

## 11. 什么时候 imagined data 值得进入主训练

**一个实用判断是**：只有当 imagined data 在关心维度上足够可信时，才值得进入主训练。常见高价值条件包括：

1. 被多模型或 ensemble 一致支持；
2. 通过规则或经典模拟器校验；
3. 和真实高价值边界样本高度相关；
4. 主要补的是真实数据明显缺失的区域；
5. 进入训练后能在 holdout 风险集上带来稳定收益。

## 12. 什么时候它更适合作评测而不是训练

**如果 imagined data**：

1. 风格很真实，但真实性不够确定；
2. 主要用于拉高压力测试难度；
3. 容易放大模型偏差；
4. 涉及高风险安全边界；

那么更稳妥的做法往往是先把它作为 stress test 或挑战集，而不是直接喂回主模型。

## 13. 一个机器人数据飞轮示例

以桌面操作为例，可以构造如下飞轮：

1. 收集真实示范、失败和恢复轨迹；
2. 训练世界模型，识别哪些状态分歧高；
3. 在这些状态上生成反事实动作候选；
4. 用规则和局部经典模拟器校验接触与碰撞；
5. 把通过校验的样本送入恢复策略训练；
6. 把高分歧样本送人工示范；
7. 下一轮继续围绕新的 near-miss 采样。

这类飞轮的重点不是 imagined data 数量，而是它是否真的推动了失败边界收缩。

## 14. 一个自动驾驶数据飞轮示例

1. 从真实行车日志中提取 near-miss；
2. 用世界模型生成不同刹车、并线、礼让策略下的反事实分支；
3. 用高保真模拟器和规则约束做复核；
4. 生成更难的风险验证集；
5. 把高价值风险样本回流到规划和风险头训练；
6. 线上或封闭测试验证是否真正降低高风险事件。

这里 imagined data 的价值，往往更多体现在风险覆盖和 stress test，而不只是主模型监督。

## 15. 常见失效模式

### 15.1 imagined data 过度主导训练

真实数据占比被挤压，系统逐渐学会了自己的幻觉分布。

### 15.2 高风险样本未经校验直接回流

模型在最危险区域形成错误自信。

### 15.3 只生成成功轨迹，不生成恢复与失败边界

系统最终只会“看起来更会成功”，却不会真正更会处理失败。

### 15.4 评价 imagined data 只看视觉或短期一致性

忽略了长时 rollout 和动作后果。

## 16. 设计建议

1. 不要把 imagined data 当作免费真实数据，而要把它当作有风险但高杠杆的辅助资源；
2. 为不同用途建立不同门禁，不要一股脑全部喂回主模型；
3. 优先围绕真实高价值失败样本做反事实扩展，而不是漫无目的大规模生成；
4. 用经典模拟器、规则、ensemble 和人工共同校验高风险场景；
5. 单独评估 imagined data 是否真的带来长尾收益、风险收益或恢复收益。

## 17. 小结

模拟器、反事实生成与合成 rollout 数据，是世界模型从“会预测”走向“会驱动下一轮学习和验证”的关键一步。真正有价值的 imagined data 不是数量大，而是能在正确的门禁下，围绕真实高价值边界，为规划、风险、恢复和数据飞轮提供新的有效样本。也只有在这种受控闭环里，世界模型驱动的 synthetic rollout 才会成为能力放大器，而不是偏差放大器。

## 18. imagined data 的一个实用判断标准

如果一类合成数据不能至少在以下一项上证明价值，就不应轻易进入主训练：

1. 显著补齐真实数据稀缺边界；
2. 提升恢复或风险专项指标；
3. 帮助构建更难的验证集；
4. 降低人工采样成本。

### 18.1 为什么这个判断很重要

因为 imagined data 一旦没有明确用途，就很容易退化为“生成很多看起来像样的数据”，却并没有真正缩小系统盲区。

## 19. 小结

反事实和合成 rollout 的关键，不在于生成能力有多炫，而在于它们是否被放进了一个受控、可验证、用途明确的闭环里。只有这样，世界模型驱动的模拟器和 synthetic 数据层，才会成为真实系统的增益，而不是新的不确定来源。
