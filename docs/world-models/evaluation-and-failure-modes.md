# 评测与失效模式

世界模型的魅力在于“先想象，再行动”。但也正因为它在内部构造了一个可供规划的未来，评测就不能只看一步预测误差。一个视频世界模型可能生成很漂亮的未来帧，却在控制上毫无帮助；一个 `RSSM` 可能重建误差不高，却在长时规划时逐步偏航。对世界模型的评测，必须同时覆盖建模质量、规划效用和安全边界。

更进一步说，世界模型的评测难点，不只是“指标很多”，而是它天然跨越了三层问题：

1. 它是不是学会了世界；
2. 它是不是学会了与动作相关的世界；
3. 它学到的东西能不能真正帮助决策和部署。

## 1. 世界模型到底在评测什么

**世界模型通常包含三个层面**：

1. **状态表示**：是否把环境压缩成足够充分的隐变量；
2. **动力学预测**：给定当前状态和动作，能否合理预测未来；
3. **决策支持**：这个模型能否帮助规划、控制、探索或评估风险。

**因此它的误差也天然分成三类**：

1. **观测误差**：生成画面、重建特征、预测传感器是否准确；
2. **状态误差**：隐藏状态是否保留任务相关信息；
3. **决策误差**：基于模型做出的规划是否比 model-free 或简单启发式更好。

很多研究只重视第一类，这会让模型看起来“会画图”，却不一定“会想事”。

## 2. WM、WAM、VAM 的评测重点并不一样

虽然这三类都属于世界模型谱系，但真正需要回答的问题不同：

### 2.1 `WM`

**重点通常是**：

1. world rollout 是否稳定；
2. 奖励、风险、终止头是否可读；
3. imagined planning 是否真能提升决策；
4. hidden state 是否支持 belief 维护。

### 2.2 `WAM`

除了 world 预测，还要问：

1. 动作和未来是否被联合建模得更好；
2. 动作生成是否真的从 world 建模中获益；
3. joint modeling 是否优于显式的 `world + policy` 分层；
4. 统一生成是否真的提高了 counterfactual 质量。

### 2.3 `VAM`

除了未来视频质量，还要问：

1. 视频先验是否真的提升动作质量；
2. 视频 latent 是否比静态视觉表征更有控制价值；
3. 在线控制时是否需要完整视频解码，还是 latent 已足够；
4. 视频分支带来的收益是否大于训练和推理成本。

所以一套统一而粗糙的“world model 指标表”往往不够。  
**更好的做法是先问**：这篇方法到底想证明什么。

## 3. 观测层评测

### 3.1 一步预测误差

**最直接的指标是一步预测损失**：

$$
\mathcal{L}_{1\text{-step}}
=
\mathbb{E}\big[
\ell(\hat{o}_{t+1}, o_{t+1})
\big].
$$

其中 \(o_t\) 可以是图像、点云、状态向量或多模态观测。 \(\ell\) 可以是均方误差、交叉熵、感知损失或离散 token `NLL`。

一步误差重要，但它的局限很明显：模型可以通过局部平滑和平均化取得不错分数，却在多步滚动时迅速失真。

### 3.2 多步 rollout 误差

给定长度为 \(H\) 的轨迹，世界模型常需评估 rollout 误差：

$$
\mathcal{L}_{\text{rollout}}(H)
=
\frac{1}{H}
\sum_{h=1}^{H}
\mathbb{E}\big[
\ell(\hat{o}_{t+h}, o_{t+h})
\big].
$$

如果 \(\mathcal{L}_{\text{rollout}}(1)\) 很好，但随 \(H\) 增大快速恶化，说明模型无法进行长期闭环推演。对机器人和自动驾驶来说，这比一步误差更关键。

### 3.3 感知质量并不等于决策质量

视频世界模型常用 `FVD`、`PSNR`、`SSIM`、`LPIPS` 等指标。这些适合评估生成质量，但不能保证规划价值。一个模型生成的未来视频很清晰，却把“车会切线”这种稀有行为洗平了；另一个模型画质一般，却保留了关键交互事件。后者更可能服务决策。

这说明世界模型评测不能只借用生成模型传统指标。

### 3.4 Horizon-conditioned 评测很重要

同一个模型在 `1-step`、`5-step`、`20-step`、`100-step` 上的行为可能完全不同。  
因此更合理的做法是报告按 horizon 分层的误差曲线，而不是只报一个平均值。

## 4. 状态表示层评测

### 4.1 充分性与最小性

理想的隐状态 \(s_t\) 需要既保留任务相关信息，又不过度记忆无关细节。可以写成一个概念化目标：

$$
\max I(s_t; y_t) - \beta I(s_t; o_t),
$$

其中 \(y_t\) 表示与任务有关的变量，如未来奖励、碰撞风险、对象位置。第一项要求充分性，第二项鼓励压缩。

实际中很难直接计算互信息，因此常用代理任务测试隐藏状态：

1. 是否可线性解码目标位置、速度、接触状态；
2. 是否可预测未来奖励和终止事件；
3. 是否可做成功/失败轨迹判别；
4. 是否能支持 counterfactual 动作区分。

### 4.2 Belief consistency

在部分可观测环境里，世界模型的隐藏状态应能整合历史证据，形成“信念状态”。例如机器人看不到抽屉内部，但根据上一步动作能推断抽屉是否已打开。若隐藏状态对长期历史不敏感，模型就会在遮挡环境里失效。

**可以构造一致性测试**：向模型提供不同观测子序列，但这些序列在物理上应对应同一隐藏状态，检查模型输出是否接近：

$$
\mathcal{L}_{\text{belief}}
=
\mathbb{E}\|s_t^{(a)} - s_t^{(b)}\|_2^2.
$$

### 4.3 干预可读性

对世界模型来说，仅仅把状态压紧还不够。还要看该状态在动作改变时，是否会产生与真实世界一致的可解释变化。也就是说，状态不仅要能“记住过去”，还要能“区分未来”。

## 5. 决策层评测

### 5.1 Planning return

最终最重要的指标之一，是用该世界模型进行规划后的累计回报：

$$
J(\pi_{\text{plan}})
=
\mathbb{E}\left[
\sum_{t=0}^{T}\gamma^t r_t
\right].
$$

这里的关键不在于绝对值，而在于比较：

1. 与无模型策略比是否有优势；
2. 与真实环境 `MPC` 或专家比差多少；
3. 在数据分布外任务中是否仍有帮助。

### 5.2 Model bias induced regret

若世界模型系统性地偏向某些未来，会把规划带偏。可以定义模型偏差导致的 regret：

$$
\text{Regret}_{\text{model}}
=
J(\pi^\star_{\text{env}}) - J(\pi^\star_{\text{wm}}),
$$

其中 \(\pi^\star_{\text{env}}\) 是在真实环境最优的策略，\(\pi^\star_{\text{wm}}\) 是基于世界模型求得的最优策略。现实里通常无法精确求这两项，但可以通过离线数据、仿真对照、有限在线试验估计。

### 5.3 Action sensitivity

对于 `WAM / VAM` 尤其重要的是动作敏感性。  
也就是说，当动作改变时，模型预测的未来是否发生了**正确且足够显著**的变化。

**可以定义一个概念性差异指标**：

$$
\Delta_{\text{act}} =
d\Big(
\hat{o}_{t+1:t+H}^{(a)},
\hat{o}_{t+1:t+H}^{(a')}
\Big),
$$

并和真实环境中动作差异导致的未来差异比较。  
如果模型对不同动作给出的未来都差不多，那它即使画面很真，也对控制没什么价值。

### 5.4 Counterfactual ranking

更进一步，可以测模型是否能正确判断两个反事实动作谁更优。  
这类测试比单纯预测误差更贴近决策，因为它直接问模型：  
**你是否能区分“更好的未来”和“更差的未来”？**

## 6. 风险与安全评测

在自动驾驶或高风险机器人里，平均回报不足以说明问题。还需测：

1. 碰撞率；
2. 越界率；
3. 急刹率；
4. 任务中断率；
5. 人工接管率；
6. 高风险场景下的不确定性校准。

若世界模型只拟合“常见正常情况”，在低概率危险事件上失真，那么平均 return 再高也不值得上线。

### 6.1 长尾风险为什么必须单独评测

训练数据里最少的，往往恰恰是部署时最重要的：

1. near-miss；
2. 罕见遮挡；
3. 规则边界；
4. 交互分支稀有事件；
5. 复杂接触失败。

因此风险评测不能只在平均数据分布上做。

## 7. Open-loop 与 Closed-loop 的区别

这是世界模型评测最容易被忽视的一点。

### 7.1 Open-loop

给定真实历史，预测未来并与真实未来比较。  
**优点是**：

1. 容易离线大规模跑；
2. 指标清晰；
3. 便于统一数据集对比。

**缺点是**：

1. 不反映模型被拿去规划后的闭环偏差；
2. 不体现错误是否会被后续动作放大。

### 7.2 Closed-loop

让世界模型真正进入策略或规划回路，再看最终行为。  
优点是更贴近真实部署；  
缺点是昂贵、复杂、变量更多。

### 7.3 为什么两者必须一起看

因为 open-loop 好不代表 closed-loop 好。  
世界模型只要有一点系统性偏差，在闭环里就可能反复放大。

## 8. Benchmark 与数据集评测不应只看单一分布

更合理的 world model benchmark 至少应区分：

1. in-distribution；
2. boundary cases；
3. domain shift；
4. rare events；
5. intervention / counterfactual splits。

### 8.1 为什么很多 benchmark 仍然不够

**因为它们只覆盖**：

1. 常见状态；
2. 真实轨迹续写；
3. 单一观测模态。

**而没有很好覆盖**：

1. 动作干预；
2. 策略利用偏差；
3. 规则和安全边界；
4. 失败恢复与 near-miss。

## 9. 失效模式一：短期拟合好，长期滚动坏

这是最典型的问题。模型在 teacher forcing 条件下一步预测很准，但 rollout 几十步后进入未见状态分布，误差快速积累。数学上，若每步误差为 \(\epsilon_t\)，动力学 `Lipschitz` 常数为 \(L\)，则多步误差可能近似放大为

$$
\|\delta_{t+h}\|
\lesssim
\sum_{i=0}^{h-1} L^i \epsilon_{t+h-i}.
$$

当 \(L > 1\) 或局部不稳定时，误差会指数式扩散。

机器人开抽屉任务里，前两帧位置略偏一点可能没事，但滚动十几步后，模型中的机械臂已经“穿过”把手；规划器在这个假世界里会给出完全不可执行的动作序列。

## 10. 失效模式二：平均化未来

当未来本身多模态时，世界模型常会预测一个“不存在但均值合理”的未来。比如路口左转车辆可能直行也可能减速等待，模型输出一条模糊、居中的轨迹。生成视频时就是模糊影子；用于规划时，则意味着重要分支被抹平。

**解决这类问题通常需要**：

1. 随机潜变量或离散 latent 表达多模态；
2. 序列扩散或 token 化生成更丰富的未来分布；
3. 在规划时进行粒子采样或分支搜索，而不是只看均值轨迹。

## 11. 失效模式三：学会视觉纹理，没学会因果约束

视频模型很擅长拟合纹理和局部运动，但世界模型真正需要的是“如果施加这个动作，会发生什么”。也就是说，需要条件因果结构，而非仅仅是观测相关性。

一个常见例子是机械臂桌面操作。模型可能准确生成“物体出现在抓手附近”的画面，却并没有真正建模抓取接触、摩擦和夹爪闭合带来的状态转移。于是当动作稍有变化，预测立刻失真。

这类问题在评测中可通过 counterfactual 动作测试暴露：保持观测不变，只改变动作序列，检查模型是否给出符合物理的分化未来。

## 12. 失效模式四：奖励或任务头可读性差

有些世界模型观测重建不错，但隐状态难以预测奖励、成功概率或约束违反事件。对决策来说，这种模型就像“画图好看，但不会打分”。可以单独训练一个 readout 测试：

$$
\hat{r}_t = g_r(s_t), \quad \hat{c}_t = g_c(s_t),
$$

检查奖励与约束预测精度。若这一步很差，说明隐藏状态没有把任务关键信息组织出来。

## 13. 失效模式五：数据覆盖不足导致幻想

世界模型非常依赖训练数据支持的状态-动作覆盖。进入训练稀疏区后，它容易自信地产生“幻觉式未来”。在自动驾驶中，罕见交通参与者行为、施工路段、突发横穿行人常属于覆盖不足区；在机器人中，透明物体、软体物体、反光表面和接触失败轨迹常是稀疏区域。

**因此评测应明确区分**：

1. 数据高密度区的性能；
2. 数据边缘区的性能；
3. 完全 `OOD` 场景的退化方式。

不是所有 `OOD` 都能解决，但至少要知道模型在这些区域是“不确定”、还是“错误但自信”。

## 14. 失效模式六：规划器学会利用模型漏洞

这是非常现实但常被忽视的问题。  
一旦世界模型被放进优化回路，规划器可能找到那些“在模型里看起来好、在真实世界里其实坏”的动作序列。

### 14.1 为什么这很危险

因为这类漏洞在 open-loop 评测里未必明显，但在规划时会被系统性放大。  
**评测上应尽量寻找**：

1. 高回报但高模型偏差区域；
2. 规划器反复选择的异常动作；
3. 模型和真实环境的 ranking mismatch。

## 15. 不确定性评测

世界模型如果要用于规划，最好能输出不确定性。设模型预测分布为 \(p(o_{t+1}|s_t,a_t)\)，可以评估：

1. `NLL` 是否校准；
2. 预测区间覆盖率是否合理；
3. 不确定性与真实 rollout 误差是否相关。

例如定义相关性

$$
\rho = \mathrm{corr}(u_t, \ell(\hat{o}_{t+1}, o_{t+1})),
$$

其中 \(u_t\) 是不确定性分数。若 \(\rho\) 很低，说明不确定性头只是装饰，不能为风险规划提供信号。

### 15.1 Calibration 比“会报不确定”更重要

一个模型天天说自己不确定，和在真正危险时才说不确定，是完全不同的能力。  
因此不确定性评测必须看校准，而不是只看数值大小。

## 16. 面向 WM、WAM、VAM 的专项验收

### 16.1 `WM` 的专项验收

**应重点看**：

1. 长时 rollout；
2. reward / done 可读性；
3. imagined planning 收益；
4. OOD 风险前瞻。

### 16.2 `WAM` 的专项验收

除了上面这些，还要问：

1. 联合生成动作和世界是否真的优于分层式 `world + policy`；
2. 动作质量的提升是否来自 world 分支，而不是更大模型容量；
3. 动作 token 和世界 token 的 joint modeling 是否带来更好的 counterfactual 区分。

### 16.3 `VAM` 的专项验收

除了视频质量和动作质量，还要问：

1. 视频先验是否真能提升控制泛化；
2. 动作解码是否必须依赖完整视频生成；
3. 在实时控制里，latent-level inference 是否已经足够；
4. 统一视频-动作训练是否比单纯 `VLA` 更省数据。

## 17. 面向机器人与自动驾驶的专项评测

### 17.1 机器人

机器人世界模型建议至少加入以下专项测试：

1. **长时接触任务 rollout**：如开门、插拔、堆叠；
2. **观测遮挡测试**：目标被手、工具或容器遮挡；
3. **动作反事实测试**：轻微改变抓取角度，看未来是否合理变化；
4. **成功率敏感区测试**：例如杯口与放置边缘只差几厘米的情形；
5. **恢复能力评测**：失败后几步内能否回到成功区。

### 17.2 自动驾驶

**自动驾驶里要关注**：

1. 多主体交互预测质量；
2. 稀有风险事件，如鬼探头、加塞、突然停车；
3. 地图条件变化，如施工、临时锥桶、车道封闭；
4. **规划闭环指标**：碰撞率、舒适性、通过效率；
5. 风险触发与接管时机。

对驾驶任务而言，离线 open-loop 轨迹误差和闭环驾驶表现经常不一致，因此必须保留闭环评测。

## 18. 一个可靠的评测流水线

**推荐把世界模型验收分成五层**：

1. **观测层**：一步、多步、感知质量、不确定性校准；
2. **状态层**：奖励可读性、信念一致性、线性探测任务；
3. **决策层**：规划回报、成功率、风险指标、闭环表现；
4. **鲁棒层**：稀有事件、域偏移、部分观测、长时任务；
5. **部署层**：实时预算、失败回退、人工接管和线上稳定性。

只有五层一起看，才能避免“视频很好看，但机器人做不成事”的误判。

### 18.1 再加一层：统一模型的收益归因

对 `WAM / VAM` 这类统一模型，还建议加一层额外分析：

1. 收益来自 world 分支还是 action 分支；
2. 来自统一 latent 还是更大模型容量；
3. 来自视频先验还是来自更多数据；
4. 来自更强生成，还是来自更合理的动作接口。

否则很难知道方法真正有效的部分是什么。

## 19. 小结

世界模型的评测难，不是因为指标太多，而是因为它天生跨越了表示、预测、动作、规划和部署多个层次。只看一步误差，会把它误判成普通预测器；只看视频质量，会把它误判成普通生成模型；只看平均回报，又会忽略风险和长尾。

真正成熟的世界模型验收，必须同时回答三件事：

1. 它是否学到了有用的世界结构；
2. 它是否对动作和反事实真正敏感；
3. 它是否能在闭环里稳定创造真实决策价值。

也只有在这样的评测框架下，`WM / WAM / VAM` 这几条路线的优劣和边界，才会真正清晰。

## 20. 评测世界模型时别忘了部署可用性

很多方法在论文表里不错，但真正进系统前还要再过一层：

1. rollout 是否足够快；
2. 不确定性是否可校准；
3. 风险分数是否能接入回退逻辑；
4. 视频分支是否值得其计算成本；
5. 高风险样本是否能被稳定复现。

### 20.1 为什么这层重要

**因为世界模型如果不能进入**：

1. 控制主环；
2. 低频风险分析环；
3. 数据引擎；
4. 验证与 stress test；

任意一条真实链路，那它再强也更像研究原型。

## 21. 小结

世界模型评测最终要从“好不好看、准不准”走到“能不能接进真实闭环”。只有把表示、预测、规划、风险、数据回流和部署可用性放到同一套验收框架里，`WM / WAM / VAM` 的价值边界才真正能被看清。

## 22. Benchmark Taxonomy 应该先分层，而不是先列名字

世界模型评测最容易出问题的地方，是把一堆 benchmark 名字堆在一起，却没有先分清它们测的是哪一层能力。

### 22.1 建议至少分成四层

1. **观测预测层**：一步、多步、视频质量、重建质量；
2. **状态表征层**：奖励、风险、接触、目标可读出性；
3. **决策规划层**：闭环成功率、回报、碰撞率、越界率；
4. **部署可用层**：实时性、稳定性、回退能力、监控与解释性。

### 22.2 为什么这能减少误判

因为一个方法可能：

1. **观测层强，但决策层弱**；
2. **闭环回报高，但风险校准差**；
3. **视频质量好，但 rollout 时间太慢**。

只看一张综合榜单，会把这些差异全部抹平。

## 23. Open-loop 与 Closed-loop 的鸿沟必须单独量化

世界模型的一大经典问题是：**离线滚动看起来不错，闭环一接就露馅。**

### 23.1 为什么会这样

1. **open-loop 总是在真实历史轨迹附近滚动**；
2. **closed-loop 会把模型错误反馈回自己的未来输入**；
3. **策略会主动探索训练中罕见的状态**；
4. **小误差经过长 horizon 和控制反馈会被放大**。

### 23.2 应该怎么测这条鸿沟

1. **固定 open-loop 指标**，如一步、多步 rollout；
2. **固定 closed-loop 任务集**；
3. **统计两者差值和退化曲线**；
4. **特别看长 horizon、风险事件和恢复场景**。

### 23.3 一个更强的信号

不是看“closed-loop 是否优于 baseline”这一点，而是看：**open-loop 提升是否稳定转移为 closed-loop 提升**。如果长期不转移，说明模型学到的可能不是任务真正需要的结构。

## 24. Counterfactual Evaluation 应该成为标配

如果世界模型声称自己理解动作后果，却从不测反事实，那验证是不完整的。

### 24.1 反事实评测至少可以问

1. **替换动作后未来是否发生合理变化**；
2. **风险场景中是否能生成不同安全等级的分支**；
3. **策略在边界决策处，模型是否对不同动作给出显著不同预测**；
4. **动作敏感性是否集中在真正相关的状态变量上**。

### 24.2 为什么它能区分真建模与伪相关

有些模型只是学会了“场景接下来通常怎么发展”，并没真正用动作做条件。反事实评测能把这种“只会顺着历史猜未来”的模型揪出来。

## 25. 校准评测不能只做一个 ECE

世界模型里的不确定性和校准，比普通分类器更复杂，因为它预测的是序列、轨迹和多模态未来。

### 25.1 更实用的校准问题

1. **高风险预测是否真的对应更高失败率**；
2. **置信区间是否覆盖真实未来**；
3. **模型的不确定性升高时，回退策略是否真的更保守**；
4. **不同场景桶里的校准是否一致**。

### 25.2 为什么只报一个标量不够

因为同一个模型可能：

1. **在常规场景校准良好**；
2. **在稀有风险场景极度过度自信**；
3. **在遮挡、传感器缺失、多智能体交互里完全失真**。

## 26. 安全评测要把 Near-miss 单独拉出来

仅统计碰撞率或任务成功率，会错过大量最有价值的边界样本。Near-miss 往往比真实事故更高频，也更适合做系统迭代。

### 26.1 Near-miss 评测可以关注

1. **最小安全间距**；
2. **接触前最后时刻的风险判断**；
3. **紧急回避是否被及时提出**；
4. **模型是否能识别“看起来没撞，但已经进入危险边缘”的未来**。

### 26.2 为什么 Near-miss 很值钱

1. **样本更多**；
2. **更容易获得人审与自动标签**；
3. **更接近可学习的风险边界**；
4. **适合用于数据引擎回流和策略改进**。

## 27. 人类评审仍然重要，但要有结构

对于高维视频世界模型，很多错误很难靠单一自动指标完全捕捉，所以人类评审依旧必要。但它不能只是“看起来不错”。

### 27.1 建议的人审维度

1. **物理一致性**；
2. **动作敏感性**；
3. **风险可见性**；
4. **长期稳定性**；
5. **是否保留任务关键信息**。

### 27.2 更好的评审组织方式

1. **按场景桶抽样**，如接触、遮挡、急转向、多人交互；
2. **对比真实未来、模型未来和反事实未来**；
3. **让标注者回答结构化问题，而不是给一个整体分数**。

## 28. Failure Mode 最该按哪几类来记

### 28.1 表示失败

模型没有把任务关键状态压进 latent，例如：

1. **把小物体丢掉**；
2. **对遮挡后目标失忆**；
3. **对接触状态编码不足**。

### 28.2 动力学失败

模型知道当前状态，却不会正确滚动未来，例如：

1. **短期准、长期漂**；
2. **速度和方向误差不断累积**；
3. **多实体交互一复杂就崩**。

### 28.3 动作条件失败

模型生成未来时对动作不敏感，或者动作影响方向错了。

### 28.4 规划失败

模型单独预测不错，但给 planner 用时仍做不出好决策，说明：

1. **cost/readout 不好**；
2. **不确定性没接进去**；
3. **rollout 预算不够**；
4. **模型与 planner 接口不匹配**。

### 28.5 部署失败

模型离线一切正常，但：

1. **太慢**；
2. **不稳定**；
3. **无法监控**；
4. **没有回退条件**。

## 29. 评测报告最好写成“验收单”而不是论文表

如果目标是进入真实系统，建议报告至少包含：

1. **模型定位**：`WM / WAM / VAM`，主要部署位置；
2. **核心输入输出接口**；
3. **open-loop 指标**；
4. **closed-loop 指标**；
5. **counterfactual 指标**；
6. **校准与风险指标**；
7. **实时性与资源占用**；
8. **已知 failure buckets**；
9. **推荐回退条件**。

### 29.1 为什么要写成验收单

论文表强调比较，系统验收强调接不接得进去。两者重心不同，不能混用。

## 30. 线上评测要把数据回流一起设计

一旦世界模型进入真实系统，评测就不该只是一组离线分数，而要进入持续数据回流。

### 30.1 线上应重点记录

1. **高不确定性样本**；
2. **高风险分支样本**；
3. **预测与真实偏差大的片段**；
4. **planner 依赖模型后失败的案例**；
5. **人工接管和回退触发事件**。

### 30.2 这会形成什么闭环

1. **高价值样本进入再训练或专项评测桶**；
2. **failure mode 不再只是描述，而是进入数据引擎**；
3. **模型迭代能围绕真实边界问题推进**。

## 31. 一份世界模型发布门槛示例

如果要给世界模型设上线门槛，可以考虑：

1. **open-loop 达到指定阈值**；
2. **closed-loop 至少不劣于现有 baseline**；
3. **counterfactual sensitivity 通过专项测试**；
4. **风险校准在高危桶中不过度自信**；
5. **实时性满足目标部署环的预算**；
6. **已建立自动回退与监控**。

### 31.1 为什么这套门槛要同时包含性能和安全

因为世界模型如果只追平均回报，很容易在边界场景中制造看不见的风险；如果只追保守，又可能完全失去决策价值。发布门槛必须强迫团队同时看两边。

## 32. 小结

世界模型评测真正要做的，是把**预测能力、动作敏感性、规划收益、风险校准和部署可用性**放到同一张验收图里。只有这样，`WM / WAM / VAM` 的优劣才不会被单一指标误导，而能被放回真实闭环系统里判断。
