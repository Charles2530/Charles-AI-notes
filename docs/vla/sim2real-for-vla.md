# VLA 的 Sim2Real

VLA 让机器人从视觉和语言直接映射到动作，看起来像是把“感知-规划-控制”压成了一个统一接口。但这也带来一个尖锐问题：如果训练数据主要来自仿真或受控采集环境，模型如何在真实世界中保持有效？Sim2Real 不是一个附加技巧，而是决定 VLA 是否能离开实验台的核心问题。

## 1. 为什么 VLA 的 Sim2Real 比传统控制更难

传统控制里的 Sim2Real 通常围绕动力学参数、控制延迟、摩擦误差展开；VLA 则把视觉外观、语言指令、行为分布和动作接口全耦合起来，因此现实差距至少来自四层：

1. **视觉差距**：光照、反光、传感器噪声、相机标定、背景杂乱。
2. **语言差距**：用户指令更口语、更歧义、更开放。
3. **动力学差距**：接触、摩擦、执行器延迟、对象物理属性。
4. **行为差距**：真实人类示范和仿真专家策略风格不同。

因此一个在仿真中“看懂指令、动作流畅”的 VLA，到真实环境中可能同时在四个层面一起偏航。

## 2. 用分布差距来理解 Sim2Real

把训练分布写成 $p_{\text{sim}}(o, l, a)$，真实分布写成 $p_{\text{real}}(o, l, a)$，则核心问题是减少二者差距。若策略为 $\pi_\theta(a|o,l)$，真实环境中的期望回报是

$$
J_{\text{real}}(\pi_\theta)
=
\mathbb{E}_{(o,l)\sim p_{\text{real}}}
\big[
R(\pi_\theta(o,l))
\big].
$$

即便 $\pi_\theta$ 在 $p_{\text{sim}}$ 上很好，只要分布迁移大，$J_{\text{real}}$ 也可能显著下降。

## 3. 视觉 Sim2Real：不仅是 Domain Randomization

### 3.1 Domain Randomization 的作用与局限

经典做法是在仿真中随机化颜色、光照、纹理、相机位置、物体材质，让模型不要过拟合某个视觉外观。形式上可视为从扰动分布 $\mathcal{T}$ 采样：

$$
\tilde{o} = t(o), \quad t \sim \mathcal{T}.
$$

它能提升外观鲁棒性，但对文档、屏幕、精细装配等任务帮助有限，因为真实差距往往不只是“颜色不一样”，而是物理接触和任务流程本身不同。

### 3.2 更有价值的是任务相关随机化

对 VLA 来说，优先应随机化那些真正影响动作决策的因素，例如：

1. 目标物体位置与姿态
2. 遮挡程度
3. 相机高度和视角
4. 关键背景干扰物
5. 末端执行器偏移

如果你只随机背景贴图，却始终把杯子放在桌面中央，模型仍然会学到强位置偏好。

## 4. 语言 Sim2Real：仿真指令往往过于干净

很多仿真训练里的指令像程序标签：“pick up the red block and place it in the blue bowl”。真实用户更可能说：

1. “把那个红色的积木放到蓝碗里。”
2. “把左边那个红块放进去。”
3. “先把红的拿起来，放到蓝色那个碗里。”

它们包含口语、省略、指代和顺序变化。若 VLA 没见过这类表达，Sim2Real 会体现在语言接口崩塌，而不只是视觉偏移。

因此文本侧也需要扩展分布，包括：

1. 模板改写
2. 多语言 / 方言 / 行业术语
3. 指代与省略
4. 歧义后澄清的数据

## 5. 动力学 Sim2Real：动作接口才是硬边界

VLA 的动作表示常被视作模型输出细节，但在 Sim2Real 中它是决定成败的关键接口。若仿真里输出的是理想末端位姿增量，真实系统却存在延迟、限速、夹爪回弹和控制周期抖动，那么同样动作 token 在现实中的语义并不一致。

设仿真动力学为 $f_{\text{sim}}$，真实动力学为 $f_{\text{real}}$：

$$
s_{t+1}^{\text{sim}} = f_{\text{sim}}(s_t, a_t), \qquad
s_{t+1}^{\text{real}} = f_{\text{real}}(s_t, a_t).
$$

如果两者差异大，策略的开环执行误差会快速累积。因此很多系统宁愿输出较高层的动作原语，也不直接让 VLA 发底层扭矩。

## 6. 数据层面的 Sim2Real：真实回流是核心

无论仿真做得多好，真实世界回流数据几乎总是必要的。常见路径是：

1. 先用仿真和公开数据预训练；
2. 在真实平台上做少量示范和保守 rollout；
3. 收集失败案例、恢复轨迹和用户纠正；
4. 继续微调或做 offline RL / imitation refinement。

这实际上把 Sim2Real 问题转化成“如何高效利用有限真实数据”。

## 7. 模型结构层面的策略

### 7.1 共享主干 + 任务适配头

共享视觉语言主干，再让动作头针对真实平台微调。这样能保留大规模预训练表示，同时降低真实数据需求。

### 7.2 显式加入状态与几何信息

纯视觉语言输入可能不足以支撑精细控制。很多系统会加入：

1. 机器人 proprioception
2. 末端位姿
3. 力/触觉
4. 深度图或点云
5. 相机外参

这些信号在仿真和现实中的对应关系更稳定，有助于减少视觉外观差距的影响。

### 7.3 分层控制

让 VLA 负责高层动作序列，低层控制器负责闭环执行。这种方式对 Sim2Real 更友好，因为高层语义往往比底层控制细节更可迁移。

## 8. 闭环修正是 Sim2Real 的生命线

仿真中很多实验默认开环执行一串动作，而真实机器人若没有闭环修正，轻微偏差就会滚雪球。VLA 系统应不断重新观察并更新动作：

$$
a_t \sim \pi_\theta(a_t \mid o_t, l, h_t),
$$

其中 $h_t$ 包含历史与任务状态。关键不是一次计划得多完美，而是每一步是否能根据新观测修正。

一个简单例子是抓杯子。仿真中杯子位置固定，策略伸手就成功；现实中杯子可能反光、位置偏一点、桌面有滑移。闭环修正能力决定了模型能否把这些小偏差吸收掉。

## 9. 评测 Sim2Real 时应关注什么

不能只看仿真成功率和真实成功率两个数字，还应拆解：

1. 成功率差距来自视觉错判还是动作偏差；
2. 失败是否集中在特定对象、光照、相机位姿；
3. 恢复能力如何；
4. 是否出现高置信度危险动作。

**可定义一个迁移差距**：

$$
\Delta_{\text{sim2real}}
=
R_{\text{sim}} - R_{\text{real}}.
$$

但更有价值的是把它按任务桶、对象桶和环境桶进一步分解。

## 10. 两类常见失败

### 10.1 视觉感知没问题，动作执行偏

模型知道要抓哪个杯子，但末端姿态或夹爪时序不对，导致碰杯或抓空。

### 10.2 动作大致合理，但任务语义错了

模型把“把红杯放到左边托盘”理解成“放到红色托盘”或“放到左边的任意区域”。这时问题在语言 grounding，不在动力学。

Sim2Real 分析必须把这两类问题分开，否则改动方向会错。

## 11. 一个具体流程：桌面收纳机器人

假设任务是“把桌上的餐具按类别放进抽屉分区”。可行的 Sim2Real 路线是：

1. **仿真中训练基础操作**：抓取、移动、放置、抽屉开关；
2. 对视觉、对象位置、抽屉阻尼、相机视角做任务相关随机化；
3. 用真实少量示范校正语言表达、视觉纹理和接触细节；
4. 部署时保留闭环观察和失败恢复；
5. 把真实失败轨迹再回流训练。

如果跳过第 3 到 5 步，只靠仿真预训练直接上线，通常很快暴露出现实分布差距。

## 12. 一个更开放的例子：家务助理

**真实家庭环境中的差距更大**：家具布局、光照、说话方式、物体种类、被打断的概率都远超仿真可覆盖范围。此时仿真更像“预备学校”，真实世界回流和保守部署才是决定成败的部分。

## 13. 设计建议

1. 把仿真当成覆盖与预训练工具，不要把它误当真实部署替代品。
2. 动作接口要与真实控制系统对齐，避免语义漂移。
3. 优先做任务相关随机化，而不是泛泛的视觉花样。
4. 真实世界中一定要保留闭环重观察和恢复机制。
5. 用失败分桶指导数据回流，别只看一个总成功率。

## 14. 小结

VLA 的 Sim2Real，本质上是在跨越视觉、语言、动力学和行为分布的复合鸿沟。仿真可以帮模型建立基本结构，真实回流让模型学会面对脏乱复杂的现实，而闭环控制与分层接口则是把这种迁移真正变得可用的工程保障。离开这些条件，VLA 很容易在 demo 中惊艳，在现实里失手。
