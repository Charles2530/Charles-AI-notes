# 论文导读总览

这一组页面不是简单列论文清单，而是回答一个更实用的问题：

**如果你现在要进入某个主题，先读什么、为什么读、读完要带走什么。**

!!! note "初学者先抓住"

    论文导读页的作用是帮你建立阅读顺序，不是替代专题正文。先用导读确定“哪几篇是主线、每篇要带走什么”，再回到专题页补机制、工程约束和失效模式。

## 为什么要有导读页

对于新主题，很多人第一反应是搜索一串论文标题，然后按年份硬啃。  
这往往低效，因为：

- 你不知道哪篇是分水岭
- 你不知道哪些工作是主线、哪些只是分叉
- 你很容易在实现细节里迷失，忽略方法之间的继承关系

导读页的作用，就是把“阅读顺序”本身结构化。

## 这些导读各自解决什么问题

### 扩散模型导读

帮助你快速建立 `DDPM -> DDIM -> solver -> distillation -> rectified` 这条主线。

### 多模态与具身导读

帮助你把对比学习、VLM、VLA、世界模型和具身部署串成连续路线。

### 训练与系统导读

帮助你回答“模型到底怎么一步步训出来，以及为什么训练论文必须和系统问题一起看”。

### 量化与高效推理导读

帮助你从部署成本和服务系统视角理解量化、cache、routing 和 speculative。

### VLM 评测导读

帮助你区分“模型答得像”“模型真的看懂了”“模型真能完成任务”这三层评测。

### 世界模型导读

帮助你从 `World Models -> RSSM -> Dreamer -> 视频世界模型` 建立前瞻与内部模拟的阅读主线。

### 世界模型与规划专题导读

帮助你进一步聚焦“世界模型怎样真正进入规划器与控制回路”。

### VLM/VLA 开源系统导读

帮助你从“代码库形态、接口与可复用性”而不是只从论文名去理解开源系统。

### 基础模型系统研究地图

帮助你把训练、推理、量化、评测、世界模型和多模态系统放到同一张研究地图里。

## 导读页怎么看

建议按以下四个问题来读每篇论文：

1. 这篇论文到底解决了什么旧问题？
2. 它改的是训练、推理，还是路径设计？
3. 它和上一代方法的核心差异是什么？
4. 它留下了哪些后来工作继续解决的问题？

如果能回答这四个问题，你读论文时就不容易陷入“只记住标题，不知道位置”的状态。

## 一种推荐的使用方式

如果你的目标是做项目而不是只读论文，建议使用“导读页 + 正文页”双层方式：

- 先用导读页建立方法地图
- 再回到各主题正文页补公式、例子和工程细节

这样读会比直接在正文里来回跳转高效很多。

## 学习路径与复现接口

论文导读建议按“问题线、方法线、工程线”三条线同时读。

| 使用场景 | 先读 | 读完要做什么 |
| --- | --- | --- |
| 刚进入一个主题 | 对应主题导读 + 主题总览页 | 写出 3-5 个核心问题和代表方法之间的继承关系 |
| 准备做项目 | 主题导读 + 系统/评测正文页 | 标出哪些论文只需跳读，哪些论文需要复现实验或查源码 |
| 准备写笔记 | [论文阅读与笔记方法](reading-notes-methodology.md) + [参考文献总表](../references/index.md) | 每篇笔记固定记录问题、方法改动、证据、失败边界和可复现资产 |
| 准备复现 | [论文复现与可复现性导读](reproducibility-and-replication-guide.md) | 明确数据、代码、指标、随机性、资源预算和最小可行复现范围 |

后续扩写不应继续单纯堆论文名，而应优先补跳读策略、复现优先级、经典误读、系统接口和不同背景读者的最短路径。
